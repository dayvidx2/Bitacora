1. SINTAXIS

Conjunto de reglas que definen cómo deben organizarse las palabras, símbolos o instrucciones en un lenguaje de programación para que la computadora pueda interpretarlas correctamente.

2. IDENTACION

Espacios o sangrías al inicio de las líneas de código que mejoran la legibilidad y permiten estructurar bloques lógicos en algunos lenguajes (ej. Python).

3. SCRIPT

Programa o secuencia de instrucciones que automatiza tareas dentro de un sistema o aplicación. Puede ser simple (unas pocas líneas) o complejo.

4. CODIGO

Conjunto de instrucciones escritas en un lenguaje de programación que indican a la computadora qué operaciones realizar.

5. CODIGO BINARIO

Sistema de representación basado en 1 y 0 que usan las computadoras para procesar datos e instrucciones.

6. ACSII

Estándar de codificación de caracteres que asigna valores numéricos a letras, números y símbolos para facilitar la comunicación entre sistemas

7. PARAMETRO

Valor o variable que se pasa a una función, método o procedimiento para personalizar su ejecución.

8. ALGORITMO

Conjunto ordenado de pasos lógicos que resuelven un problema o realizan una tarea.

9. LOGICA

Razonamiento estructurado que permite diseñar programas que tomen decisiones y sigan un flujo de ejecución.

10. API fetch

una interfaz de JavaScript que permite realizar peticiones HTTP asíncronas a un servidor para obtener o enviar datos

11. CSS

Lenguaje que define estilos y diseño visual de páginas web.

12. BOOTSTRAP

es un framework de desarrollo front-end de código abierto que proporciona herramientas de HTML, CSS y JavaScript para crear sitios web y aplicaciones web de forma rápida y sencilla.

13. MARkDOWN

Lenguaje de marcado ligero que permite dar formato a textos de forma sencilla (ej. archivos README).

14. fastAPI

Framework de Python para construir APIs rápidas y eficientes

15. API(interfaz de programacion de aplicaciones)

Conjunto de reglas y protocolos que permiten la comunicación entre diferentes aplicaciones o sistemas de software

16. WEBSOCKET

Protocolo que habilita comunicación bidireccional en tiempo real entre cliente y servidor.

17. VISUAL STUDIO CODE

Entornos de desarrollo (IDE y editor) ampliamente usados para programar en diversos lenguajes.

18. IDE(entorno de desarrollo integrado)

es una aplicación de software que ayuda a los programadores a desarrollar código de software de manera eficiente

19. GITHUB

Plataforma de colaboración y control de versiones basada en Git.

20. STACK OVERFLOW

Comunidad en línea donde desarrolladores comparten soluciones a problemas de programación.

21. DATOS

: es cualquier representación de información (hechos, cifras, símbolos) que una computadora puede interpretar y manipular para ejecutar un programa

22. INFORMACION

Conjunto de datos organizados y procesados que adquieren significado.

23. BITS

Unidad mínima de información (0 o 1) que las computadoras procesan y almacenan

24. BYTES

unidad fundamental de información que consiste en 8 bits, utilizada para representar datos, como números, caracteres, o instrucciones, y medir el tamaño de archivos y la capacidad de almacenamiento

25. PROCESADOR(CPU)

Unidad central que interpreta y ejecuta instrucciones

26. CACHE

Memoria rápida que guarda datos de uso frecuente para acelerar el acceso.

27. GHz (gigahercios)

Unidad que mide la velocidad del procesador Memoria rápida que guarda datos de uso frecuente para acelerar el acceso

28. HILOS (threads)

Secuencias de ejecución que permiten procesar varias tareas en paralelo.

29. TDP (potencia de diseño termico)

Potencia máxima de calor que un procesador puede generar y que debe disiparse.

30 SSD (unidad de estado solido)

Dispositivo de almacenamiento más rápido que los discos duros tradicionales.

31. MOTHERBOARD (placa base)

Tarjeta principal que conecta y comunica todos los componentes del computador.

32. CHIPSET

Conjunto de circuitos que gestionan la comunicación entre procesador, memoria y periféricos.

33. BIOS/UEFI

Firmware que inicializa el hardware y carga el sistema operativo

34. FUENTE DE ALIMENTACION

Proporciona energía eléctrica a los componentes del PC.

35. REFRIGERACION

Conjunto de sistemas (ventiladores, disipadores, líquidos) que mantienen la temperatura de los componentes

36. PS/2

Conector antiguo para teclado y mouse.

37. DISPLAYPORT

Se utiliza para conectar dispositivos externos

38. INTEL/AMD

Principales fabricantes de procesadores.

39. APPLE

Empresa tecnológica que desarrolla hardware, software y sistemas operativos propios

40. VPN (virtual private network)

crea una conexión segura y cifrada entre tu dispositivo e Internet, actuando como un túnel privado que protege tu actividad en línea

41. ISP (proveedor de servicio de internet)

Proveedor que da acceso a internet.

42. SSO (inicio se sesion unico)

Sistema que permite autenticarse una sola vez para acceder a múltiples servicios.

43. ENDPOINT

Dispositivo o punto de acceso que se conecta a una red (computadora, móvil, API).

44. HTTP (protocolo de transferencia de hypertexto)

es el protocolo que permite la comunicación entre un navegador (cliente) y un servidor web para transferir datos como páginas web, imágenes y videos

45. WWW (red munidal de informacion)

Sistema de documentos y recursos enlazados a través de internet.

46. CLIENTE

es el software o hardware que solicita recursos o servicios a otro programa o componente, conocido como servidor, y a menudo es el punto de interacción del usuario

47. SERVIDOR

: proceso de crear y ejecutar scripts (programas) en un servidor web para generar contenido dinámico que se entrega a los navegadores de los usuarios en forma de código (HTML, CSS, JavaScript)

48. VENTANA DE TEXTO

: Área dentro de una aplicación destinada a escribir o mostrar información

49. MODAL

Ventana emergente que requiere interacción antes de continuar.

50. POPUP

Ventana que aparece de manera automática o al ejecutar una acción

51. BOTON DE WINDOWS

Tecla en el teclado que abre el menú inicio de Windows

52. BARRA DE TAREAS

Área en Windows que muestra aplicaciones abiertas, accesos y estado del sistema.

53. AI (inteligencia artificial)

Campo de la informática que busca crear sistemas capaces de realizar tareas que requieren inteligencia humana

54. LLM (modelo de lenguaje grande)

Modelo de IA entrenado en grandes volúmenes de texto para procesar y generar lenguaje natural.

55. ALUCINACIONES (ia)

Respuestas incorrectas o inventadas generadas por un modelo de lenguaje.

56. CANVA

Herramienta digital para diseño gráfico colaborativo

57. BSOD (azul de la muerte)

Error crítico en Windows que provoca la detención del sistema

58. XML (lenguaje de marcado extensible)

Lenguaje de marcado usado para estructurar y transportar datos.

59. ACCESO DIRECTO

es un pequeño archivo que funciona como un puntero o enlace hacia otro archivo, carpeta, aplicación o recurso en la red, permitiendo un acceso rápido y sencillo a estos elementos sin tener que buscar su ubicación original

60. TECLADO

es un dispositivo, ya sea físico o virtual, que funciona como un sistema de entrada para introducir información en un aparato electrónico o musical, mediante un conjunto de teclas que al ser presionadas envían una señal para ejecutar una función

61. TECLA DE ALTERNANCIA

son las teclas de Bloq Num o Bloq Mayús, que activan o desactivan una función de forma permanente hasta que se vuelven a pulsar

62. TECLA ALT (alternar)

se utiliza para acceder a funciones o caracteres secundarios

63. MOUSE(raton)

es un dispositivo de entrada externo para computadoras que permite al usuario interactuar con la interfaz gráfica en la pantalla, moviendo un puntero, haciendo clics, arrastrando objetos y desplazándose por documentos

64. DPI (puntos por pulgada)

se refiere a la cantidad de puntos (de tinta o píxeles) que caben en una pulgada lineal y es una medida de la resolución y nitidez de una imagen, ya sea impresa o digital

65. ARCHIVOS ELECTRONICOS

: es un conjunto de datos o información, de cualquier naturaleza (texto, imagen, audio, etc.), almacenado en formato digital en un sistema informático o medio electrónico

66. CODIGO DE BARRAS

representación gráfica de datos (generalmente líneas paralelas o puntos y cuadrados) que almacena información sobre un producto, la cual es leída por un escáner para su rápida y precisa identificación y seguimiento

67. CODIFICACION

es el proceso de transformar información o datos en un formato específico y estandarizado (un código) para diversos fines, como facilitar su almacenamiento, transmisión o procesamiento

68. ESCANER

es un dispositivo que digitaliza documentos, imágenes u objetos, transformándolos en archivos digitales que pueden ser vistos, editados y almacenados en una computadora

69. SEÑALES ELECTRICAS

transmiten información en formato binario (unos y ceros) y son procesadas por circuitos electrónicos

70. MICROFONO

es un dispositivo de entrada que se usa para transformar las ondas sonoras en energía eléctrica en procesos de grabación y reproducción de sonido

71. CAMARA WEB (webcam)

captura video y audio para transmitirlos en tiempo real a través de una computadora o Internet

72. PANTALLA TACTIL

es una pantalla que permite al usuario interactuar directamente con el contenido, ya sea tocando, deslizando o arrastrando la superficie con un dedo o un lápiz óptico

73. MONITOR

es un dispositivo electrónico que funciona como una pantalla y que tiene la función principal de mostrar información visual generada por una computadora u otro dispositivo electrónico

74. IMPRESORA

es un dispositivo que convierte información digital (texto, imágenes, gráficos) de una computadora o dispositivo móvil en un formato físico, como papel, para crear copias tangibles de esos datos

75. ALTAVOCES

un dispositivo que transforma señales eléctricas en ondas sonoras para que podamos escuchar el audio de aparatos electrónicos como computadoras y teléfonos

76. PROYECTOR

es un aparato que reproduce y magnifica imágenes o videos, enviándolos hacia una superficie grande como una pantalla o una pared

77. AURICULARES

es un dispositivo que permite escuchar audio y/o comunicarse con otras personas

78. HDD (dico duro)

es un dispositivo de almacenamiento magnético no volátil, para guardar de forma persistente

79. MEMORIA USB

es un dispositivo de almacenamiento de datos portátil y compacto que se conecta a una computadora u otro dispositivo a través de un puerto USB

80. CLASE DE ALMACENAMIENTO

define las características de rendimiento, disponibilidad y, en el contexto de la programación, la duración y ubicación de la memoria de un objeto o conjunto de datos

81. TARJETA SD

es un dispositivo de almacenamiento extraíble de memoria flash que se utiliza para guardar datos digitales

82. TARJETA DE RED

es un componente de hardware que permite a un ordenador o dispositivo conectarse a una red, ya sea a través de un cable (Ethernet) o de forma inalámbrica (Wi-Fi)

83. ROOUTER

dispositivo que conecta múltiples dispositivos (como ordenadores, teléfonos y tablets) a una red local (LAN) y, a través de un módem, distribuye la conexión a Internet a estos dispositivos de forma cableada o inalámbrica (Wi-Fi)

84 Abstracción

es el proceso de simplificar sistemas complejos al ocultar los detalles internos y exponer solo las características esenciales de un objeto o función

85. Accesibilidad

es el proceso de diseñar y desarrollar software y sitios web para que sean utilizables por todas las personas

86. ACL (Lista de Control de Acceso)

es un conjunto de reglas que controlan qué usuarios o dispositivos pueden acceder a un recurso, como una red, un archivo o una aplicación

87. Adaptador de red

es un hardware que te permite enlazar a varios ordenadores entre ellos con el fin de enviar y recibir documentos, fotos o archivos, discos duros, impresoras, entre otros

88. Proyecto

son tareas prácticas que implican escribir código para crear aplicaciones de software, sitios web u otras herramientas digitales

89. Equipo

un conjunto de herramientas de hardware (computadoras, dispositivos) necesarias para programar, o un software especializado

90. Usuario

es una entidad que interactúa con un sistema, ya sea una persona (usuario final), un programa o un dispositivo, y que posee permisos y recursos asignados dentro de una red

91. Requerimiento (requisito)

es una descripción detallada de lo que un software o sistema debe hacer y las restricciones bajo las cuales debe operar, buscando satisfacer las necesidades de los usuarios y los objetivos del proyecto

92. Documento

es un archivo digital que contiene texto, datos, imágenes, audio o vídeo, de modo que incluye información que puede ser utilizada, visualizada y compartida directamente

93. Análisis

: es un proceso esencial que abarca desde la comprensión profunda de un problema y sus requisitos hasta la validación del código escrito para asegurar su eficiencia, seguridad y corrección

94. Diseño

es un proceso crucial que implica planificar y estructurar la solución de un problema informático antes de escribir el código

95. Prueba

es un proceso para verificar que un software funciona correctamente, cumple con los requisitos especificados y está libre de defectos

96. Error

: es un fallo en un programa que provoca un resultado inesperado o incorrecto, impidiendo que funcione como se espera

97. Corrección de errores

se refiere a la identificación, análisis y solución de defectos en software o sistemas informáticos

98. Versión:

es una iteración específica de un software o un conjunto de código, identificada por un número o código alfanumérico, que marca un estado particular en su desarrollo e incluye nuevas funcionalidades, correcciones de errores o cambios que la diferencian de versiones anteriores

99. Actualización (UPDATE)

es una modificación de un software, aplicación o sistema operativo que se lanza para corregir errores, añadir nuevas funciones, mejorar el rendimiento y, muy importante, para reforzar la seguridad del sistema

100. Cambio

es una modificación de un software, aplicación o sistema operativo que se lanza para corregir errores, añadir nuevas funciones, mejorar el rendimiento y, muy importante, para reforzar la seguridad del sistema

101. Tareas Programadas

son procesos automatizados que ejecutan comandos, scripts o aplicaciones en momentos específicos o bajo ciertas condiciones, sin necesidad de intervención manual del usuario

102. Entrega de SOFTWARE

que es el proceso de llevar un producto de software desde su concepción hasta que está disponible para el cliente

103. Plazo

se refiere a un intervalo de tiempo específico para completar un proyecto, una fase o una tarea, y también puede describir el proceso de organizar y distribuir ese tiempo de manera eficiente

104. Tiempo de Ejecución

es el período en el que un programa se encuentra activo y procesando instrucciones en un sistema informático, desde que se inicia hasta que finaliza su ejecución

105. Tiempo de Computo

se refiere a la duración que una computadora necesita para realizar una tarea

106. Costo

se refiere a la forma en que se calcula el coste de un proyecto, que suele basarse en las horas de trabajo por programadores de diferentes niveles

107. Presupuesto

es un plan financiero que detalla los costos estimados para completar un proyecto de software, incluyendo personal, hardware, licencias y otros gastos

108. Objetivo

es la creación de software funcional que resuelva problemas y realice tareas específicas, la obtención de programas correctos, eficientes y claros para el mantenimiento y portabilidad, y el desarrollo de soluciones innovadoras que faciliten la vida diaria en un mundo cada vez más digita

109. Resultado

es el producto de la ejecución de un programa, ya sea una salida visual en pantalla, un cambio en una base de datos, o el control de un dispositivo físico, buscando que el programa desempeñe la tarea para la que fue diseñado, resolviendo un problema o realizando una función específica

110. Proceso

es la creación de instrucciones escritas en un lenguaje de programación que una computadora puede ejecutar para resolver un problema o realizar una tarea

111. Fase

describen el proceso para crear un programa de software, que generalmente incluye el Análisis del problema, la fase de Diseño de la solución (algoritmo y estructura), la Codificación para escribir el código, las Pruebas y depuración para corregir errores, la Implementación para ponerlo en marcha y el Mantenimiento para actualizaciones y mejoras

112. Ciclo (bucle)

es una estructura de código que permite ejecutar un conjunto de instrucciones repetidamente hasta que una condición se cumpla o deje de cumplirse

113. Revisión de código

es un proceso de desarrollo de software donde los desarrolladores examinan el código para detectar errores, asegurar la calidad, la consistencia y la seguridad antes de integrarlo en la base de código principal

114. Validación

es el proceso de asegurar que los datos y el software cumplen con requisitos específicos

115. Programación Funcional

es un estilo o paradigma de programación que se centra en el uso de funciones como bloques fundamentales para construir el software, enfocándose en la aplicación de funciones matemáticas y transformaciones de datos en lugar de modificar estados o variable

116. Interfaz:

es un conjunto de reglas, protocolos y herramientas que permite la comunicación e interacción entre diferentes sistemas o componentes de software de manera estandarizada

117. Botón:

se refiere a la programación de un botón, un componente de interfaz de usuario (UI) en diversas plataformas de software, que permite a los usuarios interactuar con una aplicación mediante un clic o presión para ejecutar una acción o desencadenar un evento

118. Formulario

es un componente de la interfaz gráfica de usuario (GUI) que funciona como una ventana o un contenedor donde el usuario puede introducir datos a través de varios controles como cuadros de texto, botones y casillas de verificación

119. Menú

es un elemento de interfaz que lista opciones o comandos para que los usuarios interactúen con un programa o sistema, facilitando la navegación y el acceso a funcionalidades específicas

120. Opción

es un tipo especial que se utiliza para representar datos que pueden tener valores faltantes

121. Tabla

es una estructura de datos que organiza información en filas y columnas, similar a una hoja de cálculo

122. Registro

es una estructura de datos que agrupa varios elementos (llamados campos) de diferentes tipos, bajo un único identificador, permitiendo manejar datos relacionados de manera organizada y legible

123. Campo

un campo es una variable (o espacio de almacenamiento) que guarda un dato específico y se declara dentro de una clase o estructura

124. Archivo

es un conjunto organizado de instrucciones y datos que un programa necesita para ejecutarse y funcionar correctamente

125. Carpeta (directorio)

se utiliza para organizar y almacenar archivos o cualquier tipo de datos

126. Función

es un bloque de código reutilizable que realiza una tarea específica y puede recibir datos de entrada (parámetros) y devolver un resultado

127. Variable

es un espacio en la memoria de la computadora, al que se le da un nombre, que sirve para almacenar datos (como números, textos u otros valores) cuyo contenido puede cambiar durante la ejecución de un programa

128. Constante

es un identificador con un valor predefinido que no puede ser modificado o reasignado durante la ejecución de un programa

129. Condición

: es una instrucción lógica que evalúa si algo es verdadero o falso para determinar qué camino tomar o qué acción ejecutar dentro del programa

130. Caso de uso

describe cómo un usuario o sistema externo (actor) interactúa con un sistema para alcanzar un objetivo específico, mostrando una funcionalidad del sistema desde la perspectiva del usuario

131. Prueba unitaria

son pruebas automatizadas que verifican el correcto funcionamiento de la unidad más pequeña y aislada de código, como una función o método, asegurando que se ejecuta según lo esperado

132. Prueba funcional

son una categoría de pruebas de software que verifica que cada característica o función de una aplicación se comporte de acuerdo con las especificaciones y requisitos definidos

133. Prueba manual

son un proceso de validación de software donde los testers humanos simulan el comportamiento de un usuario final para ejecutar casos de prueba y encontrar errores, defectos y problemas en la aplicación sin el uso de herramientas de automatización

134. Prueba automática

consisten en usar software y scripts para ejecutar casos de prueba en aplicaciones de forma automática

135. Informe

es un documento digital que recopila, organiza y presenta datos de forma clara y estructurada, a menudo para analizar un proceso, un sistema, o los resultados de un proyecto

136. Reporte

es un documento o formato que organiza y presenta información, comúnmente extraída de una base de datos, de manera estructurada y fácil de interpretar para el usuario final

137. Diagrama de Flujo (flujograma)

es una representación gráfica de un proceso o algoritmo que utiliza símbolos estándar y líneas para mostrar la secuencia lógica de operaciones, decisiones y el flujo de control de un programa o sistema

138. Modelo de Software

es una representación abstracta que describe un sistema o proceso de desarrollo de software, ayudando a visualizar, planificar y comunicar el diseño de un programa o la estructura de su creación

139. Diseñador en Programación

es un profesional que crea y desarrolla soluciones de software, analizando las necesidades del usuario para diseñar programas, aplicaciones o sistemas que cumplan con requisitos específicos, utilizando principios de las ciencias de la computación e ingeniería

140. Desarrollador de Software

es un profesional que utiliza habilidades de codificación y lógica para crear, probar y mantener aplicaciones, sistemas y software de todo tipo, desde apps móviles hasta soluciones empresariales complejas

141. Programador

es un profesional que escribe, depura y mantiene código en uno o varios lenguajes de programación (como Python o Java) para crear, mejorar y solucionar problemas en software, aplicaciones y sistemas

142. Tester

es un profesional encargado de evaluar la calidad del software antes de su lanzamiento, asegurando que funcione correctamente, cumpla con los requisitos y esté libre de errores

143. Líder (Teach Lead)

es un desarrollador experimentado que no solo programa activamente, sino que también guía a un equipo de programadores para asegurar el éxito técnico y la entrega eficiente de proyectos de software

144. Analista

es un profesional que diseña, desarrolla, implementa y prueba sistemas o aplicaciones de software para resolver problemas de negocio específicos

145. Soporte

se refiere al servicio de asistencia técnica para ayudar a los usuarios a resolver problemas con un software o hardware, así como a mantener su óptimo funcionamiento a través de tareas de monitoreo, resolución de incidencias y mantenimiento

146. Documentación

es el conjunto de registros y guías que describen un software o proyecto, abarcando desde los requisitos, diseño y arquitectura hasta el código y el proceso de desarrollo

147. Guía

: es un conjunto de normas y mejores prácticas para escribir código de manera consistente y legible

148. Lista

es una estructura de datos que almacena una colección de elementos en un orden específico, permitiendo que cada elemento tenga un índice único y la posibilidad de contener elementos repetidos o de distintos tipos, según el lenguaje

149. Checklist

es una lista de tareas, requisitos o puntos de control que se utilizan para verificar la correcta ejecución de procesos, asegurar la calidad del código y evitar errores o la omisión de pasos importantes

150. Formato de código

es la práctica de organizar y estructurar el código fuente para mejorar su legibilidad, mantenibilidad y colaboración entre desarrolladores

151. Formato de Archivo

se refiere a la estructura y organización de los datos dentro de un archivo, determinando cómo se codifican, almacenan e interpretan por el software

152. Plantilla

es un modelo genérico o un esquema que permite definir estructuras de código, como funciones o clases, que operan con diferentes tipos de datos sin necesidad de reescribir el código para cada tipo específico

153. Borrador (draft)

es una versión preliminar o incompleta de un código, documento, o proyecto, que permite al desarrollador trabajar en la idea sin publicarla ni finalizarla aún

154. Versión final (lanzamiento final)

es una iteración estable y completa de un software, producto o aplicación que se considera lista para su distribución y uso por parte de los usuarios finales

155. Entrega parcial

se refiere a la acción de entregar una parte de un pedido de software o de un conjunto de tareas que se solicitó en su totalidad, dividiéndolo en envíos más pequeños o incrementales

156. Módulo

es un fragmento de código diseñado para realizar una función específica, organizado de forma independiente y que se puede reutilizar en diferentes partes de un programa o en otros programas

157. Componente

es un módulo de software independiente y reutilizable que encapsula una funcionalidad específica, actuando como un bloque de construcción para sistemas más grandes

158. Elemento

son los componentes básicos con los que se construye el código, como variables, constantes, tipo de datos, operadores e intrusiones, que cumplen una función específica dentro de un programa

159. Sistema

es un conjunto interrelacionado de elementos de software (programas y datos) y hardware (componentes físicos) que trabajan juntos para cumplir una función específica o resolver una tarea, como gestionar información o controlar un dispositivo

160. Aplicación

es un software o programa informático diseñado para realizar tareas o funciones específicas para un usuario

161. Programa

181. Observación

es el acto de recopilar información sobre cómo un sistema o sus componentes funcionan en la realidad, ya sea observando directamente a los usuarios interactuar con él o analizando su comportamiento en diferentes contextos para identificar problemas, entender necesidades y mejorar el diseño

182. Advertencia

185. Riesgo

es un valor cuantitativo que se asigna a una característica del software o del proceso de desarrollo para analizarla y tomar decisiones informadas

es una medición numérica y objetiva que evalúa características específicas de un software, un proceso de desarrollo o un proyecto

es un período de tiempo durante el cual un usuario interactúa con una aplicación o sitio web

es el proceso de transferir y configurar un software (como un programa, un sistema operativo o un controlador) en una computadora para que pueda ejecutarse y utilizarse correctamente

es la creación de una copia de los datos, archivos, configuraciones y código fuente de un proyecto para almacenarlos en un medio diferente al original

201. Restauración

es el proceso de recuperar el estado anterior de un sistema, programa o datos a partir de una copia de seguridad (o punto de restauración), que se utiliza para solucionar problemas como la pérdida de datos, errores de sistema, fallas de hardware o software no deseado

202. Desinstalación

es el proceso de eliminar un programa o aplicación de un sistema informático, borrando todos sus archivos y componentes asociados para liberar espacio y optimizar el rendimiento

es el conjunto de datos, parámetros y ajustes que determinan cómo debe funcionar un programa o sistema

204. Preferencia

es un ajuste definido por el usuario o administrador que permite personalizar el comportamiento de una aplicación, como la apariencia de la interfaz, la forma en que se muestran los datos o el modo en que se ejecutan ciertas tareas, sin alterar la lógica central del sistema

es el proceso de modificar o adaptar una aplicación, sistema o software para que cumpla con las necesidades, preferencias o especificaciones de un usuario u organización en particular

206. Tema

es un conjunto de configuraciones que determinan la apariencia y el comportamiento de un programa o sistema

es aquel cuyo estado o contenido no puede ser modificado después de su creación

208. Colaboración

es un proceso dinámico donde los desarrolladores trabajan juntos, ya sea en el mismo espacio físico o a distancia, utilizando herramientas digitales para compartir código, resolver problemas, y alcanzar un objetivo común en el desarrollo de un proyecto de software

es un conjunto de profesionales con habilidades complementarias que colaboran para desarrollar, mantener o mejorar un software, enfocados en objetivos comunes y compartiendo la responsabilidad del éxito del proyecto

proceso mediante el cual los desarrolladores dan instrucciones a una computadora para que realice tareas específicas

211. Organización del código

es la forma en que los desarrolladores estructuran, clasifican y almacenan su código para que sea más comprensible, mantenible y fácil de modificar

son las formas específicas de almacenar, organizar y manipular la información en un sistema informático para que los programas puedan acceder a ella y utilizarla de manera eficiente y rápida

213. Agenda

se refiere a una lista estructurada de tareas, actividades o eventos que deben realizarse en un momento determinado, ya sea para una reunión, un proyecto o para la ejecución de tareas automáticas dentro de un programa

es un calendario que organiza tareas, recursos y fechas de vencimiento en una secuencia ideal para que un proyecto pueda completarse a tiempo

son conjunto de datos o valores que un programa o componente tiene en un momento dado, los cuales pueden cambiar con el tiempo y afectar su comportamiento futuro

216. Cambio planeado

es la introducción intencionada y estructurada de un nuevo estado, proceso o método en el software, a menudo para mejorar la eficiencia, corregir errores o adaptarse a nuevas necesidades, siguiendo fases de diagnóstico, diseño, ejecución y evaluación para asegurar la transición y el logro de objetivos deseados

217. Cambio imprevisto

es un evento inesperado que altera el estado normal de un proyecto o sistema, como un fallo de seguridad, una interrupción del servicio o un error no anticipado, que debe ser abordado de inmediato para restaurar el funcionamiento normal, minimizando los riesgos para los usuarios y el sistema

es una estructura de control que permite al software ejecutar diferentes bloques de código dependiendo de si se cumple o no una condición lógica específica.

219. Estructura de control

son construcciones que determinan el orden en que se ejecutan las instrucciones de un programa, permitiendo que el código tome decisiones y realice acciones repetitivas en lugar de seguir un flujo lineal constante

220. Objetivo genera

es crear un software que cumpla un propósito específico, como resolver un problema, automatizar tareas o facilitar una actividad, y que sea correcto, claro, eficiente y, si es posible, portable a otras plataformas.

es una meta concreta y medible que define qué debe lograrse en una etapa particular del desarrollo o aprendizaje de un programa

es un objetivo específico y cuantificado que se ha logrado satisfactoriamente dentro de un proyecto o sistema, representando el punto final deseado y no solo un paso intermedio

223. Tiempo estimado

es una predicción o cálculo del tiempo necesario para completar una tarea o un proyecto de software, considerando factores como el alcance, la complejidad, la experiencia del equipo y los recursos disponibles

224. Tiempo real

se refiere a la capacidad de un sistema para responder a eventos o procesar datos de manera casi inmediata, cumpliendo con plazos estrictos y predecibles para garantizar la eficiencia y evitar consecuencias negativas

225. Revisión continua

es un proceso o metodología donde se revisa y valida el código o los objetivos de un proyecto de forma constante, no en intervalos fijos, para asegurar que se mantienen actualizados, relevantes y en línea con las metas establecidas

226. Revisión final

es una evaluación exhaustiva del código y el software terminado, realizada antes de que el producto sea lanzado o entregado, con el objetivo de detectar errores, defectos, problemas de seguridad y áreas de mejora, y así garantizar la calidad final del software

227. Tarea

es una unidad de trabajo o ejecución que se puede asignar a un sistema informático para que la realice

228. Etapa

se refiere a una de las fases o pasos definidos dentro del ciclo de vida del desarrollo de software, que guía el proceso desde la concepción de una idea hasta su implementación y mantenimiento

229. Iteración

es la ejecución repetida de un bloque de código hasta que se cumpla una condición específica o se alcance un número determinado de repeticiones

230. Revisión

es un proceso de examen metódico del código o de otros componentes de software para identificar errores, defectos, vulnerabilidades y áreas de mejora

es una plantilla o molde para crear objetos, la cual define las propiedades (atributos) y el comportamiento (métodos) que tendrán todos los objetos creados a partir de ella

233. Objeto

es una entidad que combina datos (propiedades) y funciones (métodos) para representar algo del mundo real o un concepto abstracto

234. Manual de usuario

es una guía detallada que explica a los usuarios cómo instalar, usar, y solucionar problemas de una aplicación o software, incluyendo sus funcionalidades, menús y procesos, utilizando un lenguaje claro y, a menudo, con capturas de pantalla y diagramas para facilitar el entendimiento

235. Manual técnico

es un documento que detalla de forma exhaustiva y para un público experto los aspectos técnicos de un software o sistema, incluyendo su arquitectura, requisitos de hardware y software, instalación, configuración, descripción de módulos, código fuente, APIs y soluciones a problemas complejos

es un modelo o versión preliminar de un producto, sistema o código que se crea para probar ideas, recopilar comentarios y validar requisitos antes del desarrollo completo

es una forma de comunicación utilizada por el sistema operativo o las aplicaciones para informar sobre condiciones, estados o errores al usuario, a otro programa, o entre distintos componentes del sistema

librería mundial online. Ofrece más de un millón de títulos a través de Internet.

240. Reproductor de DVD

es un dispositivo electrónico que reproduce tanto discos de DVD como de CD

241. Métrica de calidad

es una medida cuantitativa que se utiliza para evaluar atributos del software, como la confiabilidad, mantenibilidad o eficiencia, con el fin de entender y mejorar el proceso de desarrollo y el producto final

242. Acceso a datos

es la capacidad de un programa o usuario para recuperar, leer, modificar o transferir datos de sistemas de almacenamiento como bases de datos o archivos

243. Cuenta de usuario

es un registro de información que permite a una entidad (sea una persona o un servicio) acceder de forma segura a recursos específicos de un sistema informático, como archivos, aplicaciones o dispositivos

244. Inicio de sesión

es el proceso mediante el cual un usuario o sistema se identifica y autentica para acceder de forma segura a un sistema, aplicación, servicio o recurso

247. Copia de seguridad

248. Preferencia de usuario

son las configuraciones y elecciones que un usuario realiza para personalizar su experiencia en una aplicación o plataforma, adaptando la interfaz, la funcionalidad o el comportamiento del software a sus necesidades y gustos personales

249. Diseño visual

proceso de crear y organizar elementos gráficos para desarrollar software, donde se utilizan interfaces de arrastrar y soltar, diagramas e iconos en lugar de código textual para construir la lógica de una aplicación

250. Estructura de Datos

es una forma especializada de organizar, almacenar y manipular datos dentro de un sistema informático para permitir el acceso y la gestión eficiente de la información

es un identificador único que señala una ubicación específica en la memoria de una computadora donde se almacenan datos o instrucciones

252. Enlace (link)

es un elemento (como texto, una imagen o un botón) que, al ser activado por el usuario, lo dirige a otro contenido, ya sea en la misma página, en otra página web o a un archivo

es una acción o suceso que ocurre en el sistema, como la pulsación de una tecla o el clic de un botón, que el programa puede detectar y ante el cual puede ejecutar código de respuesta

254. Entrada (input)

es cualquier dato o instrucción que se proporciona a un programa para que pueda realizar su función, recibiendo información desde el exterior para su procesamiento interno

es la información o el resultado que un programa genera y hace disponible al usuario o a otro sistema, ya sea en la pantalla, a través de un sonido, una impresión o guardándola en un archivo

267. Operador

es un símbolo o palabra reservada que indica al compilador que realice una operación específica sobre uno o más operandos (datos o variables)

268. Cálculo

es cualquier proceso bien definido, ya sea aritmético o no aritmético, que se utiliza para resolver problemas y obtener un resultado

269. Índice

es una posición o un número que identifica de forma única un elemento dentro de una estructura de datos, como un array o una lista, permitiendo acceder directamente a ese elemento

270. Etiqueta

son palabras claves, marcas o fragmentos de código que se adjunta a un dato, contenido o elemento para organizarlo, identificarlo y describir su propósito, funcionando a menudo como metadatos o instrucciones para un intérprete

es un mensaje que se muestra a un usuario para informarle sobre un evento o información relevante de una aplicación o sistema, sin necesidad de que el usuario esté interactuando directamente con ella

301. Alerta

se refiere a una notificación automática o un aviso que indica la detección de una vulnerabilidad, un incidente de seguridad o una actividad maliciosa en el código, la aplicación o el sistema que un desarrollador está construyendo o manteniendo

302. Mensaje

es una comunicación que se envía de un objeto o componente a otro para solicitar que realice una acción, similar a una petición o una instrucción

303. Cuadro de confirmación

es un cuadro de dialogo modal que solicita al usuario que tome una decisión binaria (generalmente "Aceptar" o "Cancelar") sobre una acción que puede tener consecuencias importantes o no ser fácilmente reversible

304. Éxito

es cuando se logra combinar habilidades técnicas como el aprendizaje continuo de lenguajes, la escritura de código limpio y la gestión de versiones, con habilidades blandas como la comunicación, el pensamiento crítico, la creatividad y el trabajo en equipo

305. Progreso

se refiere tanto al avance en el desarrollo de un proyecto (como el trabajo completado de un total previsto) como a la mejora del propio desarrollador a través del aprendizaje y la prácticav

306. Indicador

es una variable lógica que se utiliza para registrar si ha ocurrido un evento o estado específico dentro de la ejecución de un programa

307. Localización de software

es el proceso de adaptar una aplicación o un programa a las necesidades lingüísticas, culturales, técnicas y normativas de un mercado específico, más allá de la simple traducción

308. Rol

se refiere al puesto, función o conjunto de permisos asignados a un usuario o entidad dentro de un sistema de software o proyecto, definiendo sus tareas, responsabilidades y los recursos a los que tiene acceso

309. Perfilado de software

es una técnica de análisis dinámico que se usa para medir el comportamiento de un programa mientras se ejecuta, identificando cuellos de botella, áreas de código ineficientes y el uso de recursos como el tiempo de CPU o la memoria

310. Programación de tareas

se refiere a la automatización de la ejecución de programas, scripts o procesos a intervalos de tiempo específicos o al ocurrir ciertos eventos

311. Token de software

verifica la identidad de un usuario para acceder a un recurso protegido o ejecutar una acción, actuando como una llave o credencial cifrada para una aplicación

312. Contraseña temporal (TAP)

es cuando se verifica la identidad de un usuario para acceder a un recurso protegido o ejecutar una acción, actuando como una llave o credencial cifrada para una aplicación

313. Verificación de código

es un proceso crucial en desarrollo de software que implica revisar el código para detectar errores, asegurar la calidad, la conformidad con estándares y compartir conocimientos entre desarrolladores

314. Autenticación

es el proceso que verifica la identidad de un usuario, servicio o aplicación para confirmar que es quien dice ser, antes de permitirle acceder a un sistema o recurso digital

315. Privacidad

se refiere a proteger la información personal y sensible de los usuarios al diseñar, desarrollar y operar software y sistemas

316. Seguridad

es la práctica de desarrollar software resistente a ataques maliciosos, implementando mecanismos de protección desde la concepción del proyecto hasta su despliegue y mantenimiento

317. Protección

se refiere a un conjunto de medidas técnicas y organizativas para salvaguardar los activos digitales (como el software, los datos y las aplicaciones) contra el acceso no autorizado, el uso indebido, la pérdida o la piratería

318. Cifrado

proceso de codificar datos para que solo las personas con la clave correcta puedan descifrarlos y leerlos, protegiendo la información confidencial de accesos no autorizados

319. Desencriptado

es el proceso de convertir datos que previamente fueron cifrados de vuelta a su formato original y legible

320. Firma de función

es una descripción de cómo interactuar con una función, definiendo su nombre, los tipos de los parámetros que acepta (su entrada) y el tipo de dato que devuelve (su salida)

321. Firewall

es un componente de software o hardware que actúa como una barrera de seguridad para proteger una red o un sistema individual del tráfico no autorizado, filtrando los datos entrantes y salientes según reglas preestablecidas

322. Filtro

es un mecanismo que selecciona y muestra o procesa solo aquellos elementos de un conjunto de datos que cumplen con un criterio específico, dejando fuera el resto

323. Bloqueo

es una situación en la que dos o más procesos quedan detenidos permanentemente, porque cada uno necesita un recurso que está siendo retenido por otro proceso del grupo, creando un ciclo de dependencia circular y esperando indefinidamente por la liberación del recurso

324. Desbloqueo

se refiere a eliminar una restricción que impide que un proceso, recurso o dispositivo funcione, sea accesible o se ejecute según lo esperado

325. Verificación

es el proceso que asegura que el software se ha desarrollado correctamente y cumple con las especificaciones y requisitos definidos en las etapas iniciales del desarrollo

326. Estilo de codificación

es un conjunto de reglas y convenciones para escribir y formatear código fuente, con el fin de hacerlo más claro, legible y mantenible

327. Tamaño

es la cantidad de memoria que ocupa una variable, estructura de datos o un programa en sí mismo, medido en unidades como bytes

328. Espaciado

son los espacios en blanco (espacios, tabulaciones, saltos de línea) que no tiene un significado ejecutable, pero es crucial para la legibilidad y estructura del código, y también a las propiedades de estilo visual como el interlineado o el espaciado entre párrafos y caracteres, que son configuraciones usadas en lenguajes de marcado y diseño para mejorar la presentación

329. Iconografía en informática

se refiere al uso de iconos, que son pictogramas visuales para representar archivos, carpetas, programas y acciones en un sistema operativo o aplicación

330. Jerarquía de software

es la organización de los elementos de un sistema informático en diferentes niveles o capas, donde cada nivel se encarga de funciones específicas y está organizado según su importancia, complejidad o abstracción

331. Consistencia

es mantener la uniformidad y coherencia en todos los aspectos del desarrollo de software, desde el diseño de la interfaz de usuario hasta el estilo del código y la integridad de los datos de una base de datos

332. Adaptabilidad

es la capacidad de un sistema, software o desarrollador para ajustarse y responder de manera efectiva a cambios en requisitos, entornos, tecnologías y desafíos, sin comprometer su funcionamiento o integridad

333. Portabilidad

es la capacidad de un software para ser transferido y ejecutado en diferentes entornos, como sistemas operativos, arquitecturas de hardware o plataformas distintas, con cambios mínimos en su código original

334. Escalabilidad

es la capacidad de un software para ser transferido y ejecutado en diferentes entornos, como sistemas operativos, arquitecturas de hardware o plataformas distintas, con cambios mínimos en su código original

335. Rendimiento

se refiere a qué tan bien un programa usa los recursos del sistema (como CPU, memoria y red) y qué tan rápido ejecuta sus tareas

336. Optimización

es el proceso de modificar un programa para que funcione de forma más eficiente, utilizando menos recursos (como tiempo de ejecución o memoria) o para que alcance un rendimiento mejorado, ya sea por un menor consumo de energía o por la velocidad de su ejecución

337. Eficiencia

se refiere a qué tan bien un algoritmo o código utiliza los recursos computacionales disponibles, principalmente el tiempo de ejecución y la memoria (espacio), para resolver un problema

338. Confiabilidad

es la probabilidad de que un sistema o software funcione correctamente y sin fallos durante un período de tiempo específico y en un entorno definido

339. Estabilidad

se refiere a la capacidad de un sistema para funcionar de manera consistente y predecible, manteniendo su comportamiento y salida bajo control, sin fallar, congelarse o exhibir un comportamiento inesperado ante diversas entradas y cargas

340. Integración

es el proceso de conectar y unificar software, datos, aplicaciones y dispositivos para que funcionen juntos como un sistema cohesivo, mejorando la eficiencia, la productividad y la agilidad

341. Interoperabilidad

es la capacidad de diferentes sistemas, aplicaciones o lenguajes de programación para comunicarse, intercambiar datos y funcionar de manera coordinada, incluso si son de tecnologías distintas

342. Conversión

es el proceso de transformar un valor de un tipo de dato a otro

343. Sincronización

es el proceso de coordinar y ordenar la ejecución de múltiples procesos o hilos para asegurar que accedan a recursos compartidos de manera consistente y controlada

344. Recuperación

es el proceso de restaurar datos, un sistema o una funcionalidad que se ha perdido, corrompido o vuelto inaccesible debido a un fallo, error o interrupción

345. Arranque inicial

es el proceso que pone en marcha un sistema informático cargando el sistema operativo y otros programas esenciales en la memoria (RAM) para que el hardware pueda ser utilizado y el usuario pueda interactuar con él

346. Exportación

la acción de sacar o transferir datos y archivos de una aplicación a otro formato o sistema para que puedan ser utilizados en otro lugar

347. Importación

es la acción de incorporar código, funciones, clases o módulos definidos en un archivo o biblioteca externa a tu propio archivo de código

348. Guardado

: es el proceso de almacenar de forma permanente los datos o cambios realizados en un programa o archivo en un dispositivo de almacenamiento como un disco duro

349. Eliminación

se refiere a borrar información o código, ya sea archivos, registros de bases de datos, o incluso tareas programadas, mediante una acción específica de un usuario, un programa o el sistema operativo

350. Edición de código

es el acto de escribir, modificar y gestionar el código fuente de un programa utilizando un editor de código

351. Visualización

consiste en usar código para crear representaciones gráficas de información compleja, como gráficos, tablas y mapas, con el fin de hacerla más fácil de comprender, identificar patrones y tendencias, y comunicar resultados de manera efectiva

352. Impresión

es la acción de mostrar información, ya sea en la consola (para la depuración y el estado del programa)

353. Descarga

es la acción de recibir o transferir datos y archivos desde un servidor remoto a un dispositivo o computadora local, creando una copia de ese contenido en el equipo propio

354. Programa base

es el software esencial que controla el hardware y permite que otros programas funcionen

355. Aplicación simple

es un sistema o entorno de software que sirve como base para desarrollar, ejecutar y gestionar aplicaciones

356. Aplicación compleja

se refiere a un sistema de software que, a diferencia de una aplicación simple, integra funcionalidades de múltiples aplicaciones o módulos autónomos, o bien, presenta un gran número de interacciones internas

357. Sistema básico

referirse a los conceptos fundamentales para construir cualquier programa (como algoritmos, variables y estructuras de control) o a lenguajes de programación simples como el BASIC

358. Sistema avanzado

es el desarrollo de software utilizando paradigmas y herramientas complejas como la programación funcional, orientada a objetos, frameworks, y patrones de diseño para crear aplicaciones eficientes, escalables y seguras

359. Sistema Avanzado de Planificación y Programación (APS)

es un software que ayuda a las empresas de producción en sus esfuerzos por supervisar y planificar la producción integrando así todos los recursos

360. Plataforma digital

es un marco tecnológico integrado de hardware y software que proporciona un entorno estandarizado donde se desarrollan y ejecutan aplicaciones, y donde múltiples componentes pueden interactuar

361. Framework

es un conjunto predefinido de herramientas, librerías y reglas que proporciona una estructura y un esqueleto básico para desarrollar aplicaciones de software

362. Biblioteca

es una colección de código (como funciones, clases y objetos) prescrito y reutilizable que los desarrolladores usan para simplificar el desarrollo de software y evitar crear funciones comunes desde cero

363. Paquete

es una colección organizada de elementos relacionados, como clases, interfaces y funciones, que se agrupan en una estructura jerárquica para mejorar la organización, la reutilización y la gestión del código

364. Módulo compartido

es una unidad de código reutilizable que contiene elementos comunes (como componentes, directivas o funciones) y se puede usar en diferentes partes de una aplicación o en múltiples proyectos, evitando así la duplicación de código y fomentando la organización y eficiencia

365. Componente visual

es un elemento de la interfaz gráfica de una aplicación que el usuario puede ver y con el que puede interactuar, como un botón, un cuadro de texto o una ventana

366. Componente lógico

es una parte del software de un sistema informático, a diferencia del hardware (los componentes físicos), que realiza tareas específicas y puede ser un sistema operativo, una aplicación, un controlador o un entorno de desarrollo

367. Interfaz gráfica

es un medio visual que permite a los usuarios interactuar con programas o sistemas operativos mediante elementos gráficos como iconos, botones y menús, en lugar de comandos de texto

368. Interfaz web

es el componente de un sitio o aplicación web que permite la interacción entre el usuario y el sistema, a través de elementos visuales y funcionales como botones, menús y formularios, que se acceden mediante un navegado

369. Interfaz móvil

es el conjunto de elementos visuales e interactivos en un dispositivo móvil (smartphones y tablets) con los que los usuarios interactúan para usar una aplicación o el sistema operativo

370. Formulario de entrada

es una interfaz gráfica compuesta por campos de texto, botones y otros controles que permiten a los usuarios ingresar, modificar y enviar datos de forma estructurada a una aplicación o sistema, facilitando la interacción con la información de manera organizada y precisa.

371. Botón principal (mouse)

se refiere generalmente al botón izquierdo del ratón, que es el que se utiliza para realizar acciones como seleccionar objetos, activar elementos de la interfaz gráfica (GUI) y ejecutar funciones en las aplicaciones

372. Botón secundario (mouse)

se refiere generalmente al botón derecho es el que desencadena la aparición de menús contextuales con opciones específicas para el elemento bajo el puntero, como cortar, copiar o ver propiedades

373. Caja de búsqueda

es ese recuadro estrecho, situado normalmente en la parte superior de la página, que permite buscar contenidos en un sitio web

374. Área de texto

es un elemento de interfaz gráfica de usuario (GUI) que permite a los usuarios introducir texto de varias líneas, como comentarios o descripciones largas

375. Lista simple

es una estructura de datos que organiza elementos en una secuencia lineal a través de nodos conectados, donde cada nodo contiene un dato y una referencia al siguiente nodo

376. estructura de control condicional múltiple

permite ejecutar un bloque de código específico entre varias opciones, basándose en el valor de una única variable o expresión

377. Selector (HTML)

sirve para crear un menú desplegable (lista combinada) que permite a los usuarios seleccionar una o más opciones de una lista predefinida

378. Casilla de verificación (checkbox)

es un componente de la interfaz gráfica de usuario que permite al usuario seleccionar o deseleccionar una o más opciones de una lista, o activar/desactivar una función específica

379. Campo vacío

se refiere a un valor que representa la ausencia de contenido o datos. Puede significar una cadena vacía (como "")

380. Campo oculto

es un elemento de un formulario que no es visible para el usuario final, pero que almacena datos que se envían al servidor o se utilizan en el fondo para el funcionamiento de la aplicación

381. Etiqueta de texto (HTML)

es un código que se utiliza para definir y estructurar el contenido de un documento, indicando al navegador cómo debe mostrar o interpretar ese texto

382. Barra superior

se refiere a la barra de título de una aplicación, que contiene el nombre de la aplicación y los controles para minimizar, maximizar y cerrar la ventana

383. Barra inferior

es un componente de interfaz de usuario móvil que contiene íconos y texto para acceso rápido a las secciones principales de una aplicación, permitiendo a los usuarios cambiar entre vistas de alto nivel con un solo toque

384. Barra lateral

es un panel vertical ubicado a un lado de la ventana principal de una aplicación o sitio web, que se utiliza para mostrar información adicional, navegación o funciones que complementan el contenido principal

385. Menú secundario

es una lista de opciones que aparece de forma dependiente del contexto, a menudo como resultado de hacer clic con el botón derecho en un objeto de la interfaz, o como una sub-categoría dentro de un menú principal

386. Menú oculto

se refiere a menús o funcionalidades que no son visibles o accesibles directamente por el usuario, pero que pueden activarse mediante una combinación de teclas, una acción específica, o a través de un ítem de menú no listado

387. Submenú desplegable

es una lista secundaria de opciones que aparece al interactuar con un elemento de un menú principal, generalmente al hacer clic o pasar el cursor sobre él

388. Ícono de inicio

es una representación gráfica de un programa, una función o un archivo que facilita al usuario iniciar esa aplicación o acceder a esa opción desde la pantalla de inicio o la barra de tareas del sistema operativo, sin tener que escribir su nombre

389. Ícono de usuario

es una representación gráfica abstracta, usualmente la silueta de una persona, que se utiliza dentro de una interfaz gráfica de usuario (GUI) para identificar o representar a un usuario dentro de un sistema, ya sea como inicio de sesión o en un perfil de usuario

390. Ícono de configuración

generalmente se refiere a un icono que representa la aplicación o la funcionalidad de "Configuración" dentro de una interfaz gráfica de usuario de un sistema operativo o aplicación

391. Ícono de ayuda

es una representación visual, usualmente un signo de interrogación, que al hacer clic en él, muestra el sistema de ayuda de una aplicación, brindando información sobre el uso, la funcionalidad o la solución de dudas de la aplicación o de un lenguaje de programación específico

392. Ícono de alerta

es una pequeña ventana emergente que muestra un mensaje o advertencia al usuario, o a un elemento visual (como un símbolo de triángulo con un signo de exclamación) que acompaña a mensajes de advertencia dentro de la interfaz de una aplicación

393. Ventana principal

es la interfaz gráfica de usuario (GUI) principal y el punto de partida de una aplicación, que actúa como el centro de control para todas las demás funciones, módulos y ventanas secundarias

394. Ventana emergente

es una pequeña ventana que aparece repentinamente en un sitio web o aplicación para mostrar información adicional, solicitar una acción al usuario o presentar publicidad, sin que el usuario cierre la ventana principal

395. Ventana flotante

es una ventana gráfica que aparece sobre la ventana principal de una aplicación o página web, sin cerrarla, para mostrar información adicional, solicitar datos o confirmar una acción

396. Pestaña activa

es la que el usuario tiene seleccionada y visible en una interfaz de usuario, ya sea un navegador web o una aplicación, para interactuar con su contenido

397. Pestaña inactiva

se refiere a una ventana o pestaña del navegador que no está en uso activo y que los navegadores (como Chrome o Edge) ponen en estado de suspensión para ahorrar memoria y CPU, liberando recursos para las pestañas activas y mejorando el rendimiento general del sistema.

398. Panel lateral

es un elemento de la interfaz gráfica de usuario (GUI) que se muestra en un lado de la ventana de una aplicación para proporcionar información adicional, navegación o funcionalidad sin ocupar el área principal de trabajo

399. Panel superior

se refiere a una sección en la parte superior de una ventana o pantalla que contiene elementos de control o información, como un menú de aplicaciones, herramientas o un área para organizar otras ventanas o contenido

400. Panel inferior

es un elemento de la interfaz gráfica de usuario que se ubica en la parte inferior de la pantalla y contiene herramientas para lanzar aplicaciones, cambiar entre ventanas, acceder al menú de inicio, mostrar la hora y otras funcionalidades del sistema operativo o de una aplicación específico

401. Vista genera

se refiere a una representación o visualización de datos, código o de la estructura de una aplicación que permite ver la información de forma resumida y organizada

402. Visualización de datos

consiste en transformar datos complejos en representaciones gráficas como gráficos, mapas y paneles de control, utilizando elementos visuales para facilitar su comprensión, análisis de patrones y comunicación de hallazgos

403. Pantalla dividida

es una técnica que consiste en dividir el área visible de una interfaz en varias secciones, donde cada sección puede mostrar una aplicación o contenido diferente simultáneamente

404. Pantalla completa

es un modo de visualización que hace que una aplicación o contenido ocupe toda el área de la pantalla del dispositivo, eliminando las barras de herramientas, menús y otros elementos de la interfaz de usuario para ofrecer una experiencia inmersiva y sin distracciones

405. Tema claro

se refiere a una interfaz de usuario que utiliza un fondo de color (generalmente blanco o gris claro) y texto oscuro para su visualización, similar a un documento impreso

406. Tema oscuro

es una funcionalidad que cambia el esquema de colores de una interfaz de usuario, pasando de fondos claros a oscuros y textos claros a tonos más suaves

407. Estructura de condición simple

es un control de flujo en programación que ejecuta un bloque de código solo si una condición booleana (que evalúa a verdadero o falso) se cumple

408. Elemento interactivo

es un componente de una interfaz gráfica de usuario (GUI) que permite a los usuarios realizar acciones e interactuar con el contenido de un programa o página web

409. Elemento pasivo

es un componente de un sistema o código que realizan funciones auxiliares o secundarias, sin ser el centro de control principal ni generar la acción

410. Acción rápida

es una herramienta que permite realizar tareas complejas o repetitivas con una sola operación, como refactorizar código, generar componentes, o aplicar modificaciones en bloque.

411. Acción lenta

se refiere a la lentitud con la que se ejecuta un código o se carga una página web

412. Acción completada

es una instrucción que ejecuta un proceso o una tarea específica, como una asignación de variable

413. Evento disparado

son señales que indican que algo ha sucedido en un sistema (como un clic del usuario o una acción del sistema), y el código responde a estos eventos para realizar una acción específica

414. Evento cancelado

es la acción de detener o anular un evento previamente programado en un sistema o aplicación

415. Resultado correcto

s la salida o valor que un programa entrega y que coincide exactamente con lo que se esperaba que hiciera según las especificaciones previas a su desarrollo

416. Resultado incorrecto

es la consecuencia de un error o fallo en el código, que hace que el software produzca un resultado inesperado o erróneo, en lugar de funcionar como se había planeado

417. Prueba de humo

es una verificación rápida de la funcionalidad básica y crítica de una nueva versión (compilación) de software, para asegurar que las características esenciales funcionan correctamente antes de realizar pruebas más exhaustivas

418. Prueba final de datos

es el proceso de verificar la precisión, la calidad y la integridad de los datos antes de que se utilicen en un sistema o aplicación

419. Error crítico

es un defecto de software que causa una falla importante, un comportamiento inesperado o una interrupción en el funcionamiento de un programa o sistema, impidiendo que se complete una operación vital o que se ejecute como se espera

420. Error menor

generalmente se refiere a un error de sintaxis, que es un problema en la escritura del código que impide su correcta compilación o ejecución

421. Fallo lógico

es un defecto en la secuencia de instrucciones de un programa que provoca resultados incorrectos o inesperados, a pesar de que el código se compila y ejecuta sin fallos técnicos

422. Error de sintaxis

ocurre cuando se viola la gramática del lenguaje, resultando en código incorrecto que el intérprete o compilador no puede entender o procesar, impidiendo la ejecución del programa

423. Notificación activa

es un mensaje que una aplicación envía al usuario para informarle sobre eventos importantes o alertas cuando la aplicación no está en primer plano

424. Notificación inactiva

es una alerta o mensaje que se ha programado para no mostrarse inmediatamente o durante un período de uso activo, sino en un momento más adecuado para el usuario

425. Advertencia mostrada

es una notificación generada por herramientas de desarrollo (como el compilador o el intérprete) que indica un problema potencial o un comportamiento inesperado en el código, aunque no detiene la ejecución del programa

426. Advertencia ignorada

es cuando el programador decide explícitamente deshabilitar o ignorar un mensaje de advertencia generado por el compilador o la herramienta de análisis, generalmente porque sabe que el problema potencial que indica la advertencia no es crítico para el funcionamiento de su programa

427. Entrada válida

es un dato que cumple con todos los criterios de formato, tipo, longitud y seguridad que una aplicación o función espera antes de ser procesado

428. Entrada inválida

se refiere a datos o instrucciones que no cumplen con el formato, el tipo o las restricciones esperadas por el programa

429. Salida esperada

es el resultado que un programa debe producir para cumplir con su objetivo, generalmente mostrando información en la pantalla

430. Salida inesperada

se refiere a un error que causa que un programa termine abruptamente, se comporte de forma incorrecta o arroje resultados no previstos

431. Carga rápida

es un sistema que permite recargar la batería de un dispositivo a una velocidad mucho mayor que la carga estándar, logrando una carga considerable en menos tiempo mediante el aumento de voltaje y amperaje

432. Carga lenta

es una técnica para aplazar la carga de un objeto, contenido o recurso hasta el momento en que realmente sea necesario para la aplicación o página web

433. Arrastrar y soltar

En entornos gráficos de usuario, consiste en apuntar con el ratón o dispositivo señalizador a un determinado objeto, arrastrarlo mediante la presión constante del botón izquierdo del ratón hasta su nueva posición, donde, al dejar de hacer presión en el botón, se situará el objeto.

434. Indicador de progreso

es un componente visual que muestra el estado de una operación al usuario, ya sea mostrando cuánto falta para que se complete (determinado) o que simplemente se está trabajando en algo (indeterminado)

435. Indicador de error

es un mensaje de un compilador o herramienta de análisis que alerta sobre un problema potencial en el código, pero que no impide su ejecución

436. Signos sintácticos

es un símbolo específico (como punto y coma, corchetes, paréntesis, etc.) que tiene una función definida y debe usarse de acuerdo con las reglas de la sintaxis de un lenguaje para que el código sea correcto y entendible por la computadora

437. Estado inicial

es la primera condición o configuración de un sistema, proceso u objeto, que se representa como un punto de partida antes de que ocurra cualquier cambio o acción significativa

438. Estado activo

se refiere a un proceso o dispositivo que está en uso o ejecutándose activamente, ya sea utilizando un procesador, realizando una operación o respondiendo a una entrada

439. Estado inactivo

se refiere a un proceso o sistema que está ejecutándose pero que no realiza ninguna tarea o procesamiento activo, esperando una entrada del usuario, de otro programa, o simplemente no teniendo nada que hacer en ese momento

440. Registro de actividad

es un archivo o conjunto de datos que guarda un historial detallado de eventos, acciones, errores y cambios dentro de un sistema, aplicación o proyecto, anotando la fecha y hora exactas de cada suceso

441. Historial de acciones

es un registro detallado y cronológico de todas las operaciones, cambios o eventos significativos que se han realizado en un sistema, documento o aplicación, e incluye información como quién realizó la acción, cuándo la realizó, y el resultado de la misma

442. Historial de cambio

es un sistema que registra todas las modificaciones realizadas a un proyecto de software, permitiendo a los desarrolladores y equipos ver quién realizó cada cambio, cuándo y qué se modificó

443. Archivo temporal

es un archivo creado para almacenar datos de forma provisional mientras un programa se ejecuta, liberando memoria o como medida de seguridad para evitar la pérdida de datos

444. Carpeta oculta

se refiere a un archivo de directorio que está configurado para no mostrarse al usuario a menos que se solicite explícitamente

445. Directorio raíz

es el directorio de más alto nivel en la jerarquía de un sistema de archivos, funcionando como el punto de partida para todos los demás archivos y carpetas

446. Directorio interno

se referirse al directorio actual o de trabajo donde se ejecutan las operaciones, o a los directorios que contienen la propia estructura del programa, como las carpetas que alojan el código fuente y los recursos necesarios para su ejecución

447. Ruta absoluta

es la ubicación completa y única de un archivo o directorio dentro de un sistema de archivos, comenzando desde la raíz del sistema hasta el archivo o directorio deseado

448. Ruta relativa

es la ubicación de un archivo o directorio en relación con un punto de referencia, generalmente el directorio de trabajo actual

449. Acceso permitido

es una regla que autoriza a un usuario, programa o proceso a realizar una acción específica sobre un recurso (como archivos, datos o funciones) dentro de un sistema

450. Modificador de accesos

es una palabra clave en programación que especifica la visibilidad y accesibilidad de los miembros de una clase (como campos, métodos y constructores), controlando qué otras partes del código pueden interactuar con ellos

451. Acceso público

se refiere a un modificador de acceso que permite que un elemento de código (como una clase, método o variable) sea accesible y utilizable desde cualquier otra parte del programa

452. Perfil privado

se refiere a la característica de un miembro de una clase (como un atributo o un método) de ser inaccesible desde el exterior de esa clase

453. Rol de usuario principal

es el más alto nivel de autoridad y control, otorgando permisos para gestionar cuentas de otros usuarios, configurar ajustes del sistema, y ejecutar tareas administrativas cruciales para el funcionamiento y la seguridad de la plataforma

454. Permiso completo

se refiere a la autorización total para que un programa o usuario pueda acceder a todos los recursos o funcionalidades de un sistema

455. Permiso limitado

es un conjunto de autorizaciones que restringe las acciones que un usuario o una aplicación pueden realizar dentro de un sistema o software. En lugar de otorgar acceso total, este tipo de permiso solo permite ciertas operaciones específicas, como la visualización de datos o la interacción con elementos concretos, pero no la modificación o el control completo del sistema.

456. KeyError

generalmente se refiere a un error lógico o a un error de sintaxis que provoca que un programa no funcione como se espera, o incluso que no se ejecute

457. Token válido

es la unidad más pequeña y significativa de un lenguaje de programación (como una palabra clave, un identificador, un operador o un símbolo)

458. Token inválido

ocurre cuando un token (una cadena de texto con un propósito específico) no es reconocido o aceptado por el sistema, ya sea porque está expirado, es incorrecto, el formato es erróneo, la firma está corrupta o no se corresponde con el token esperado

459. Autorización aceptada

ocurre cuando un sistema verifica que un usuario o entidad ha sido concedido el permiso para acceder a un recurso o realizar una acción específica

460. Autorización rechazada

ocurre cuando un sistema o servicio niega acceso a un usuario o proceso porque no tiene los permisos necesarios para realizar una acción o acceder a un recurso

461. Protección alta

es un enfoque para crear software robusto y a prueba de ciber amenazas, integrando prácticas de seguridad en todo el ciclo de vida del desarrollo para minimizar vulnerabilidades desde el diseño hasta el despliegue

462. Protección baja

técnicas de bajo nivel para gestionar la memoria y los datos de forma eficiente y segura, como a la implementación de medidas de seguridad dentro del código y en el software para prevenir ataques y vulnerabilidades

463. Conexión estable

se refiere a una conexión que se mantiene constante y confiable, sin interrupciones, pérdidas de paquetes o fluctuaciones significativas de velocidad

464. Conexión inestable

se refiere a una interrupción o variación errática en la comunicación entre dos sistemas, como un servidor y un cliente, lo que causa fallos, retrasos o pérdida de datos

465. Aplicación de escritorio

es un software que se instala y ejecuta localmente en una computadora, utilizando sus recursos y sin depender de un navegador web para su funcionamiento, aunque puede conectarse a Internet y bases de datos externas

466. Programa ligero

se refiere a una aplicación o componente de software que es simple, eficiente, y utiliza una cantidad mínima de recursos de hardware (memoria, procesamiento, etc.) y software (dependencias, código) para funcionar

467. Programa pesado

es aquel que, por su diseño y complejidad, exige muchos recursos de hardware para funcionar, como procesador, memoria RAM y almacenamiento

468. Software libre

es aquel cuyo código fuente se pone a disposición para que los usuarios puedan usarlo, estudiarlo, modificarlo y distribuirlo libremente

469. Software de pago

es cualquier programa informático para el cual se debe pagar un precio para su uso, ya sea a través de una licencia de compra única, una suscripción o un modelo de pago por uso. Estos programas, a diferencia del software libre, tienen un acceso restringido a su código fuente y su distribución es controlada por el propietario o desarrollador.

470. Software educativo

es una herramienta digital diseñada específicamente para enseñar y aprender conceptos de programación. Estos programas facilitan la construcción de conocimientos sobre programación de forma autónoma e interactiva

471. Software empresarial

es una aplicación o sistema informático diseñado para satisfacer las necesidades de las organizaciones, automatizando, gestionando y optimizando procesos complejos como contabilidad, recursos humanos, logística y ventas. Estos programas buscan aumentar la productividad, mejorar la toma de decisiones y centralizar la información en un solo lugar

472. Sistema operativo

s el software base que actúa como intermediario entre el hardware de la computadora y el resto del software, gestionando los recursos del sistema (CPU, memoria, etc.) y proporcionando una plataforma para que los programas de aplicación se ejecuten

473. Entorno gráfico

es un conjunto de elementos visuales y un sistema de interacción para que el usuario pueda controlar y usar un programa o sistema operativo sin necesidad de escribir comandos de texto

474. Entorno web

es el conjunto de tecnologías y herramientas que permiten crear, mantener y probar aplicaciones y sitios web en línea, proporcionando un espacio seguro para el desarrollo

475. Entorno móvil

se refiere al conjunto de dispositivos, sistemas operativos (como Android e iOS), herramientas de desarrollo y lenguajes que se utilizan para crear y ejecutar aplicaciones en teléfonos inteligentes y tabletas

476. Entorno de pruebas

es un espacio seguro y controlado, a menudo una réplica del entorno de producción, que se utiliza para probar una aplicación de software en busca de errores y validar su funcionamiento antes de su lanzamiento final a los usuarios

477. Entorno de producción

es el sistema en el que el software está desplegado, se ejecuta y está disponible para los usuarios finales reales, gestionando datos reales y operaciones en vivo

478. Entorno loca

es la configuración de tu computadora personal (hardware y software) para desarrollar, probar y depurar aplicaciones, antes de desplegarlas en un entorno en vivo o de producción

479. Entorno remoto

se refiere al acceso y control de sistemas, datos y herramientas de desarrollo desde una ubicación física distante, utilizando software de escritorio remoto, VPNs y herramientas de colaboración para trabajar de forma segura y eficiente en el código y aplicaciones

480. Nube

se refiere a la entrega bajo demanda de recursos informáticos a través de Internet, incluyendo servidores, almacenamiento, bases de datos, redes y software

481. Servidor remoto

es una aplicación o proceso que se ejecuta en un servidor diferente al de la aplicación principal y a la cual se accede a través de una red (como Internet) para realizar una función específica, como acceso a bases de datos, servicios web, o manejo de colas de mensajes

482. Servidor local

es un entorno de desarrollo y pruebas que se ejecuta en tu propio ordenador, permitiéndote ejecutar y probar sitios web y aplicaciones sin estar conectado a Internet

483. Cliente web

es una aplicación de software o un dispositivo que inicia solicitudes de información o servicios a un servidor y recibe y procesa la respuesta del servidor para el usuario

484. Cliente móvil

es una aplicación o software diseñado para ejecutarse en dispositivos móviles (teléfonos inteligentes, tabletas) que solicita y recibe servicios o datos de un servidor remoto

485. Cliente de escritorio

es una aplicación de software que se instala en el ordenador de un usuario y se comunica con un servidor para solicitar y procesar información o servicios

486. Base de datos

es un conjunto organizado de datos almacenados electrónicamente en un sistema informático, que puede ser gestionado por un Sistema de Gestión de Bases de Datos (SGBD) para facilitar su acceso, actualización, gestión y consulta por parte de aplicaciones y usuarios

487. Tabla de datos

es una estructura organizada en filas y columnas que sirve para almacenar, gestionar y presentar información de forma sistemática, similar a una hoja de cálculo

488. Registro único

referirse a un registro de datos con un identificador único (como un UUID o un número de registro de base de datos) que garantiza que cada entrada sea distinta de todas las demás, o a la función de un programa o herramienta para extraer solo las entradas no repetidas de un conjunto de datos

489. Consulta avanzada

es una petición a una base de datos o sistema para extraer, manipular o analizar datos de forma compleja, utilizando técnicas que van más allá de las operaciones básicas

490. Filtro de búsqueda

es una lógica o un conjunto de criterios utilizados para refinar los resultados de una consulta, reduciendo el volumen de datos y mostrando solo aquellos que cumplen condiciones específicas

491. Orden ascendente

significa organizar elementos (números, texto, fechas, etc.) de menor a mayor, alfabéticamente de la A a la Z, o cronológicamente del más antiguo al más reciente.

492. Orden descendente

es la disposición de elementos en una secuencia de mayor a menor. Esto significa que cada elemento es menor o igual al elemento que le precede en la lista o colección

493. Clave primaria

es un campo (o un conjunto de campos) en una tabla de una base de datos que identifica de forma única cada fila o registro. Sus características principales son que no puede tener valores nulos y no puede tener valores duplicados en la misma tabla

494. Clave secundaria

es un campo o conjunto de campos en una tabla de base de datos que puede usarse para identificar de forma única un registro, pero que no fue seleccionado como la clave primaria principal

495. Relación uno a uno

es una conexión entre dos tablas o entidades donde cada registro en la primera tabla puede estar asociado a un único registro en la segunda tabla, y viceversa. Esta relación se utiliza comúnmente para dividir una tabla muy grande, aislar datos sensibles o almacenar información que solo se aplica a un subconjunto de la tabla principal, haciendo que la base de datos sea más eficiente y organizada.

496. Relación uno a muchos

es una conexión entre dos entidades o tablas, donde una única instancia de la entidad "uno" puede estar asociada a cero o más instancias de la entidad "muchos", mientras que cada instancia de la entidad "muchos" solo puede estar relacionada con una única instancia de la entidad "uno”

497. Relación muchos a muchos

describe una situación donde una entidad puede estar asociada con múltiples entidades de otro tipo, y viceversa. Para implementarla en una base de datos, se utiliza una tabla intermedia (o de enlace), que contiene claves foráneas de ambas tablas originales para registrar las asociaciones entre sus registros.

498. Índice de búsqueda

Es el medio por el cual se organizan y estructuran los datos para que los motores de búsqueda puedan generar resultados relevantes

499. Vista rápida

se refiere a una función del sistema operativo para ver el contenido de un archivo sin abrir el programa asociado (como "Vista Rápida" en macOS o la activación del panel de vista previa en Windows)

500. Vista previa de datos

es una herramienta que permite a los desarrolladores y analistas ver un subconjunto de datos de una tabla o fuente de datos para inspeccionar su contenido, calidad y estructura antes de usarlos completamente en un análisis o proceso de carga

501. Informe corto

Un informe que se genera de forma automática y periódica a partir de una base de datos o sistema de programación

502. Gráfico de barras

es una visualización que usa barras rectangulares para representar y comparar datos categóricos, donde la altura o longitud de cada barra es proporcional al valor o frecuencia de su categoría correspondiente

503. Gráfico de líneas

es una visualización que representa puntos de datos conectados por segmentos de línea, mostrando cómo cambian las variables a lo largo de un rango continuo, típicamente el tiempo. Sirve para identificar tendencias y patrones en los datos, lo que permite a los desarrolladores y analistas entender la evolución de métricas o procesos y tomar decisiones informadas, especialmente al analizar datos en series temporales o comparaciones continuas.

504. Gráfico circular

es una representación visual de datos que divide un círculo en porciones o sectores para mostrar la proporción de cada categoría respecto al total. Se utiliza para ilustrar relaciones de parte a todo y composiciones de datos, donde el tamaño de cada segmento es proporcional a la cantidad que representa, facilitando la comprensión de distribuciones porcentuales.

505. Gráfico de dispersión

en programación se usa para visualizar la relación entre dos variables numéricas, representando cada par de datos como un punto en un plano cartesiano con el eje X para una variable y el eje Y para la otra. Esta herramienta es fundamental en el análisis de datos para identificar correlaciones, tendencias y patrones, permitiendo a los desarrolladores o analistas de datos comprender si una variable afecta a otra y la fuerza de esa relación.

506. Panel de usuario

es un componente de la interfaz gráfica que organiza y muestra información relevante y herramientas para el usuario final dentro de una aplicación

507. Panel de administración

es una interfaz de usuario segura y centralizada para gestionar el contenido y el funcionamiento de una aplicación o plataforma web, permitiendo realizar tareas como la gestión de usuarios, contenidos, configuraciones y el monitoreo del sistema, a menudo funcionando como un "backend" privado para los responsables del proyecto.

508. Panel de control

se refiere a una interfaz gráfica de usuario (GUI) que permite a los programadores y usuarios gestionar y configurar diversos aspectos de un sistema, como el hardware, software, red o datos

509. Panel de reportes

es una interfaz visual que consolida y muestra métricas y datos clave de uno o varios procesos en tiempo real, utilizando gráficos, tablas y otras visualizaciones para ofrecer una visión general de la información más importante de forma fácil de comprender. Su objetivo es permitir un seguimiento y análisis rápido del rendimiento, la supervisión del estado de un proyecto

510. Panel de estadísticas

es una interfaz gráfica que organiza y muestra información relevante sobre el rendimiento o la actividad de una aplicación o sistema, utilizando visualizaciones como gráficos y tablas para facilitar la comprensión de métricas clave de un vistazo

511. Formulario de inicio

es una interfaz gráfica (una ventana o pantalla) que permite a los usuarios ingresar datos, como credenciales de inicio de sesión, y que se utiliza para autenticar y personalizar la experiencia dentro de una aplicación o sitio web. Su diseño, la selección de los elementos de la interfaz, la validación de los datos y su implementación en lenguajes como HTML para la web o C# y Visual Basic para aplicaciones de escritorio son fundamentales para crear una experiencia de usuario segura y eficiente.

512. Formulario de registro

es un elemento en una aplicación o sitio web que recopila información de los usuarios a través de campos de texto, casillas, botones, etc., para permitirles crear una cuenta, suscribirse a servicios, o acceder a contenido restringido

513. Formulario de contacto

es un componente de una página web que permite a los usuarios enviarles mensajes, consultas o datos directamente al administrador del sitio a través de campos de texto, como nombre, correo electrónico y mensaje

514. Campo de texto

es un elemento de interfaz gráfica de usuario en un área rectangular que permite al usuario ingresar, visualizar y procesar texto para que sea utilizado por un programa o aplicación

515. Campo numérico

es un espacio de datos diseñado para almacenar exclusivamente números, ya sean enteros o decimales, que pueden ser utilizados para operaciones aritméticas, mediciones o análisis cuantitativos. Su propósito principal es recopilar y organizar información numérica, como cantidades, edades, precios o cualquier otro dato medible, permitiendo al programa realizar cálculos o mostrar datos de forma ordenada

516. Campo de correo

es un área en una interfaz o base de datos donde se puede ingresar o almacenar una dirección de correo electrónico. Se utiliza en formularios para que los usuarios ingresen sus correos, o en sistemas de gestión para almacenar datos de contacto.

517. Botón para enviar

es un control interactivo en una interfaz de usuario que, al ser pulsado, transmite los datos ingresados por el usuario a un servidor para su procesamiento o almacenamiento

518. Botón de guardar

es un elemento de interfaz gráfica de usuario que, al ser presionado, ejecuta código para almacenar los datos ingresados o los cambios realizados en un objeto o formulario, típicamente en una base de datos o archivo

519. Botón de borrar

es un elemento interactivo en una interfaz de usuario que, al ser presionado, ejecuta una acción para eliminar o limpiar información

520. Botón de limpiar

es un elemento interactivo en una interfaz gráfica que, al ser presionado, elimina o resetea el contenido de los campos o controles de un formulario o aplicación, dejándolos en un estado vacío o inicial para permitir al usuario ingresar nuevos datos o comenzar una nueva operación

521. Botón de imprimir

es un control interactivo en una interfaz gráfica que, al hacer clic en él, ejecuta una acción de impresión, ya sea para enviar un documento o una pantalla a una impresora física, guardarla como archivo PDF, o mostrar la información en una consola

522. Botón de descargar

es un elemento de interfaz de usuario que permite a los usuarios iniciar la transferencia de un archivo desde un servidor remoto a su dispositivo local, o descargar contenido especificado por la aplicación web o del sistema

523. Botón de cargar

es un control de interfaz gráfica de usuario que indica una operación en curso o que se va a realizar, mostrando una animación o indicador de carga para informar al usuario mientras el sistema procesa una acción, como al cargar más contenido

524. Botón de buscar

es un elemento de interfaz gráfica que, al ser presionado, activa el código asociado a él para iniciar una búsqueda de información específica dentro de una aplicación o sitio web, generalmente tras introducir términos en una barra de búsqueda o campo de texto. Este botón puede aparecer como un icono de lupa o como un botón de texto y es esencial para que los usuarios encuentren contenido de manera eficiente.

525. Botón de editar

es un control interactivo en una interfaz de usuario que permite al usuario activar la función de edición, lo que puede implicar cambiar el modo de edición, modificar datos o ejecutar una acción específica. Su función es permitir al usuario interactuar con la aplicación para realizar acciones, como modificar el contenido de una página web o un documento

526. Botón de copiar

es un control en una interfaz de usuario que, al ser activado por el usuario, transfiere un dato específico (como texto, una URL o un código) al portapapeles del sistema, permitiendo que se pegue en otra aplicación o lugar

527. Botón de pegar

: es un elemento de la interfaz de usuario que permite al usuario insertar contenido (texto, imágenes, etc.) que ha sido previamente copiado o cortado y almacenado en el portapapeles del sistema. Al hacer clic en este botón, el contenido seleccionado se transfiere desde el portapapeles a la ubicación actual en la aplicación o entorno de desarrollo

528. Botón de cortar

generalmente se refiere a la función de cortar texto o elementos, que elimina el elemento seleccionado del lugar original y lo guarda en el portapapeles para poderlo pegar en otro sitio, utilizando el atajo de teclado Ctrl+X (en Windows)

529. Botón de aceptar

es un control interactivo en una interfaz gráfica de usuario (GUI) que permite al usuario confirmar una acción o aceptar la información proporcionada, cerrando una ventana o formulario y aplicando los cambios realizados

530. Botón de cancelar

se refiere a un control de interfaz de usuario que permite al usuario interrumpir una operación o salir de una acción en curso sin confirmarla, devolviéndolo al estado anterior

531. Menú principal

es una interfaz visual que lista opciones para que el usuario acceda a las diferentes funcionalidades y módulos de un software o sistema. Proporciona una navegación estructurada, como se ve en el Menú de Inicio de Windows o en la barra de menús de una aplicación, permitiendo al usuario seleccionar comandos y configuraciones para interactuar con el programa.

532. Menú contextual

es un menú emergente que aparece al realizar una acción secundaria sobre un elemento de la interfaz de usuario (como un clic derecho) y que ofrece opciones o comandos relevantes para ese elemento en ese momento específico

533. Menú desplegable

es un componente de interfaz gráfica de usuario que muestra una lista de opciones al hacer clic o pasar el cursor sobre un botón, ahorrando espacio y mejorando la organización visual de una aplicación o sitio web

534. Menú fijo

es un elemento de la interfaz gráfica que permanece visible en la pantalla y sigue al usuario mientras este se desplaza por el contenido de una página web o aplicación

535. Barra de herramientas

es un elemento de la interfaz gráfica de usuario (GUI) que contiene una fila de botones, iconos o menús desplegables que proporcionan acceso rápido a las funciones y comandos más usados de una aplicación. Permiten a los usuarios realizar tareas con eficiencia y pueden ser personalizadas para adaptarse a sus necesidades, mejorando la interactividad y la productividad en el software.

536. Barra de navegación

es un componente de la interfaz de usuario que facilita la navegación a través de un programa o sitio web, generalmente contiene enlaces a diferentes secciones, páginas o funcionalidades

537. Barra de estado

es un área horizontal, usualmente en la parte inferior de una ventana de aplicación, que muestra información sobre el estado actual de la aplicación, el progreso de tareas o mensajes contextuales al usuario, como el estado de una operación o la ubicación del curso

538. Barra de desplazamiento

es un elemento de la interfaz gráfica de usuario (GUI) que permite al usuario navegar a través de contenido que es más grande que el área visible de una ventana o contenedor

539. Ventana modal

es un cuadro de diálogo secundario que se superpone a la ventana principal de una aplicación o página web, bloqueando la interacción con el contenido de fondo hasta que el usuario la cierra

540. Pestaña abierta

generalmente se refiere a un archivo de código o un documento que se muestra en la interfaz de un entorno de desarrollo (IDE) o un editor de texto, permitiendo al programador alternar entre diferentes tareas sin abrir nuevas ventanas separadas

541. Pestaña cerrada

se refiere a una pestaña de una aplicación o ventana de navegador que ha sido eliminada o que se ha cerrado. Cuando una pestaña está cerrada, deja de estar activa y, por lo general, se pierde el contenido que tenía abierto en ella

542. Interfaz principal

es la pantalla inicial, o la interfaz visual principal, donde el usuario interactúa por primera vez con el programa a través de elementos gráficos como botones, menús, iconos y ventanas, y que sirve para iniciar o ejecutar otras funciones

543. Aplicación instalada

es un programa de software que se ha copiado y configurado en un sistema informático para su uso, almacenando archivos de programa y datos localmente para facilitar el acceso del sistema operativo. El proceso de instalación incluye la copia de archivos, la creación de directorios

544. Aplicación en línea

es un programa de software que se ejecuta en un navegador web y se accede a través de internet, sin necesidad de instalación en el dispositivo del usuario. Almacena datos y procesa información en servidores remotos (en la nube) y envía una copia temporal al dispositivo del usuario, permitiendo el acceso desde cualquier lugar y dispositivo con conexión a internet.

545. Programa educativo

es una metodología y un conjunto de herramientas que usan software para enseñar y aprender conceptos de programación, con el fin de desarrollar habilidades como el pensamiento lógico, la resolución de problemas y la creatividad

546. Programa empresarial

es un software diseñado por y para organizaciones, no usuarios individuales, que tiene como objetivo automatizar y optimizar procesos, mejorar la eficiencia, aumentar la productividad y facilitar la toma de decisiones en áreas como finanzas, logística, recursos humanos o ventas

547. Programa interactivo

es aquel que permite una comunicación de dos vías entre el usuario y el software, donde el usuario puede ingresar datos u órdenes y el programa responde en tiempo real, modificando su comportamiento o resultado. Esta interacción puede incluir acciones como hacer clic, tocar, deslizar o escribir, permitiendo al usuario influir activamente en la experiencia y el contenido del programa.

548. Sistema pequeño

se referiré a un programa simple que realiza tareas específicas, a un lenguaje de programación con una sintaxis y un conjunto de características reducidos, o a un enfoque de desarrollo donde se crean prototipos rápidamente o se hacen tareas sencillas en períodos cortos. Se caracteriza por ser fácil de entender, rápido de desarrollar

549. Sistema grande

es una aplicación de software de gran envergadura que requiere un diseño y gestión cuidadosos para manejar su complejidad y los cambios que surgen durante su desarrollo. Se caracteriza por su tamaño, que abarca múltiples subsistemas y una lógica compleja que debe mantenerse coherente.

550. Módulo básico

es un archivo o unidad lógica que contiene código (funciones, variables, clases) relacionado, permitiendo dividir un programa grande en partes más pequeñas y manejables para mejorar la organización, reutilización y el mantenimiento del código

551. Módulo avanzado

se refiere a la profundización en el concepto de modularidad (división de un programa en componentes independientes y reutilizables) y la aplicación de técnicas y conceptos de programación que elevan la complejidad y la sofisticación de los proyectos

552. Funciones rápidas

referirse a los atajos de teclado o teclas de acceso rápido, que son combinaciones de teclas que permiten ejecutar comandos o realizar tareas de forma más eficiente sin usar el mouse

553. Función lenta

se refieren a funciones que tardan demasiado en ejecutarse debido a ineficiencias en su lógica, como algoritmos complejos o bucles anidados que manejan grandes cantidades de datos

554. Proceso manual

es aquel en el que la ejecución, gestión o pruebas requieren una intervención humana directa, sin la automatización de software o sistemas

555. Operación programada

se refiere a la definición de qué tarea debe realizar un operador en un lenguaje de programación, el cual se encarga de ejecutar acciones específicas en los datos para obtener un resultado. El término "programar" una operación se usa en el contexto de la planificación de actividades, donde se secuencian y asignan recursos a cada tarea para alcanzar un objetivo.

556. Evento controlado

es el resultado de un controlador de eventos, una función que se ejecuta en respuesta a una acción o "evento" que ocurre en el programa. Este evento puede ser una interacción del usuario, como un clic de ratón, o una señal del sistema, como un error

557. Evento inesperado

es una excepción o un error que ocurre durante la ejecución de un programa y rompe el flujo normal de las instrucciones, lo que puede causar que la aplicación se detenga si no se gestiona adecuadamente. Estos eventos pueden deberse a diversas causas, como errores de programación, problemas de hardware

558. Acción pendiente

se refiere a una tarea o actividad que debe completarse pero que aún no ha sido finalizada. Este término se utiliza comúnmente en la gestión de procesos y el desarrollo de software para indicar elementos de una lista de verificación de actividades necesarias para alcanzar un objetivo

559. Acceso autorizado

significa que a un usuario o componente se le han otorgado los permisos necesarios para acceder a un recurso o realizar una acción específica dentro de un sistema, después de que su identidad haya sido verificada con éxito

560. Acceso denegado

es un mensaje de error que indica que un usuario o programa no tiene los permisos necesarios para realizar una acción específica, como acceder a un archivo, carpeta, servicio o recurso

561. Clave segura

es una combinación de caracteres (letras mayúsculas y minúsculas, números y símbolos) que protege información valiosa en sistemas o cuentas, diseñada para ser difícil de adivinar o descifrar por atacantes

562. Clave débil

es aquella que, al usarse con un algoritmo, facilita que este falle o sea inseguro, a diferencia de una clave fuerte que garantiza la seguridad

563. Código válido

es una secuencia de instrucciones que un programa de computadora puede entender y ejecutar correctamente, siguiendo las reglas de un lenguaje de programación específico. Las características clave de un código válido son su legibilidad, corrección sintáctica (que cumple las reglas del lenguaje) y funcionalidad (que realiza lo que se espera)

564. Código inválido

no es un error de sintaxis como un error de tipo, sino que se refiere a un código que no se puede ejecutar correctamente o que no produce el resultado esperado debido a condiciones de error que surgen durante la ejecución. Puede ser porque la sintaxis es incorrecta, como al escribir mal una instrucción, o porque la lógica del programa es defectuosa, resultando en resultados inesperados o fallos al correr el programa.

565. Archivo abierto

es aquel cuya especificación técnica está publicada y disponible para cualquier persona, liberándolo de restricciones legales y económicas de uso. Esto permite que cualquier desarrollador cree o lea archivos en ese formato sin costo ni dependencia de una empresa específica, garantizando así el acceso a largo plazo a la información, la interoperabilidad y la competencia en el mercado.

566. Archivo cerrado

es un archivo que ya no está siendo utilizado activamente por el programa y se ha desconectado del sistema operativo, liberando así memoria y recursos del sistema y evitando la corrupción de datos

567. Archivo comprimido

es un archivo cuyo tamaño se ha reducido utilizando algoritmos para eliminar redundancia y reorganizar datos, lo que permite ahorrar espacio en disco, acelerar transferencias de red y agrupar múltiples archivos en uno solo

568. Carpeta raíz

es la carpeta principal de un proyecto o sitio web, desde la cual se organizan todos los demás archivos y carpetas, actuando como el punto de partida de la estructura jerárquica

569. Carpeta secundaria

es una carpeta que está contenida dentro de otra carpeta, creando una estructura jerárquica para organizar archivos y código de un proyecto. Las subcarpetas dividen el trabajo en categorías más manejables, lo que facilita la localización de archivos específicos, mejora la organización del código y la gestión de proyectos grandes

570. Documento digital

es un archivo creado, almacenado y utilizado en formato electrónico que contiene información, como texto, imágenes o datos, y que puede ser visualizado y compartido en dispositivos electrónicos como computadoras o teléfonos inteligentes. A diferencia de un documento físico en papel, no requiere de un soporte material para existir y ofrece ventajas como la búsqueda instantánea y el fácil acceso desde cualquier lugar.

571. Imagen insertada

es un archivo de imagen que se incorpora en un documento o interfaz de software para su visualización o manipulación, utilizando código para referenciar su ubicación

572. Imagen eliminada

se refiere a un archivo de imagen que ha sido borrado del sistema de almacenamiento, pero no se borra permanentemente de inmediato; primero se marca como eliminado y se puede recuperar hasta que sus datos sean sobrescritos por nuevos archivos. El proceso de eliminar un archivo simplemente marca el espacio que ocupa como libre, haciendo que la imagen sea inaccesible pero todavía presente físicamente hasta que esa zona de memoria sea reutilizada

573. Video cargado

se refiere al proceso de transferir un archivo de video digital desde un dispositivo local a un servidor remoto, como una plataforma de alojamiento de videos (YouTube) o un sistema de gestión de medios

574. Video descargado

un archivo de video digital que ha sido copiado desde una computadora o servidor remoto a un dispositivo local, como un ordenador o teléfono

575. Audio reproducido

se refiere al proceso de decodificar datos de audio digital y convertirlos en ondas sonoras audibles a través de altavoces o auriculares, ejecutado por el software o hardware del sistema. La reproducción de audio implica el uso de funciones y métodos específicos en lenguajes de programación para manipular y emitir sonidos, ya sean grabados previamente o generados por el propio programa.

576. Holograma

imagen tridimensional creada por proyección fotográfica

577. Texto largo

Es un tipo de dato utilizado para almacenar grandes bloques de texto, como descripciones o comentarios extensos, que superan el límite de caracteres permitido por los tipos de texto corto. A diferencia de estos, los campos de texto largo en bases de datos como Microsoft Access pueden almacenar hasta 1 gigabyte de información

578. Texto corto

es un campo de datos que se utiliza para almacenar una cantidad limitada de información textual, como nombres, direcciones o números de teléfono, típicamente hasta 255 caracteres alfanuméricos. Aunque el nombre puede variar según el sistema o lenguaje de programación, la función principal es la misma: reservar un espacio para cadenas de texto breves que no necesitan el gran formato de almacenamiento

579. Cadena cifrada

es la versión ilegible de un texto original (texto plano) que ha sido transformada mediante un algoritmo criptográfico y una clave secreta, haciendo que su contenido sea incomprensible para quien no posea la clave correcta

580. Número entero

es un tipo de dato que representa valores sin parte decimal, es decir, números completos como -3, 0, 1, 2, 3, etc., y pueden usarse para contar, indexar, identificar o realizar cálculos matemáticos. Estos se almacenan típicamente en formatos de 32 o 64 bits en la memoria de la computadora y se les conoce comúnmente como "integers" o "int" en la mayoría de los lenguajes de programación

581. Número decimal

es un valor que puede representar partes fraccionarias de un todo, utilizando el sistema numérico de base 10 que conocemos en la vida cotidiana (dígitos del 0 al 9). Los lenguajes de programación manejan estos números mediante tipos de datos específicos, como float, double o decimal, que se utilizan para realizar cálculos precisos y almacenar valores con cifras después del punto decima

582. Registro diario

es un archivo o conjunto de datos que almacena información sobre eventos, actividades y operaciones que ocurren en un sistema o aplicación a lo largo del tiempo. Su propósito principal es documentar el funcionamiento, rastrear cambios, depurar errores y proporcionar un historial que facilita la gestión, el análisis y la recuperación de datos o del estado del sistema.

583. Registro mensual

es un tipo específico de registro que se enfoca en recopilar y presentar datos de un mes completo, como la asistencia de empleados o los ingresos y gastos de una empresa, sirviendo como un informe o un conjunto de datos estructurados para un período específico

584. Bitácora automática

es un registro electrónico que el propio sistema genera y almacena para documentar eventos, actividades, errores o cambios de manera cronológica y automática, sin intervención manual. Su propósito es ofrecer un historial detallado que permite monitorizar el rendimiento, detectar problemas, auditar acciones, y mejorar la trazabilidad y el mantenimiento del software.

585. Bitácora manual

es un registro físico, usualmente en papel, que documenta actividades, procesos o eventos importantes relacionados con un proyecto o sistema de software. A diferencia de las bitácoras electrónicas, no requiere tecnología y se utiliza para llevar un historial detallado de observaciones, avances, incidencias y soluciones de forma tradicional, aunque puede ser más susceptible a errores

586. Historial de usuario

es una descripción breve y sencilla de una característica de software, escrita desde la perspectiva del usuario final y que explica el qué, el quién y el porqué de una función deseada. Es una herramienta fundamental en las metodologías de desarrollo ágil

587. Historial corto

Registro limitado o resumido de eventos, acciones o cambios recientes realizados en un sistema, aplicación o repositorio. Generalmente muestra solo los últimos elementos del historial completo, y se usa para consultas rápidas o seguimiento inmediato.

588. Historial completo

Registro detallado de todas las acciones, modificaciones o eventos que han ocurrido dentro de un sistema, aplicación o proyecto de software. Puede incluir cambios en el código, actualizaciones de datos, actividad de usuarios, versiones del sistema y más. Es fundamental para el seguimiento, control de versiones, auditorías, mantenimiento y mejora continua del software.

589. Resultado esperado

es el comportamiento y los productos específicos que un sistema, aplicación o función debe producir bajo ciertas condiciones, tal como se define en los requisitos y especificaciones del proyecto

590. Resultado inesperado

es una anomalía o comportamiento anómalo que se produce cuando un programa de software no funciona como se espera, causando un resultado o acción incorrecta, indeseada o ilógica

591. Prueba simple

una prueba unitaria, que verifica el funcionamiento correcto de un pequeño bloque de código aislado (como una función o un método), asegurando que haga lo que se espera en cada entrada y salida específica

592. Prueba compleja

Salida de un programa o proceso que involucra múltiples valores, estructuras de datos o información elaborada, en lugar de un simple dato o valor único. Puede incluir objetos, listas, gráficos, o cualquier tipo de datos que requieran procesamiento adicional para su interpretación.

593. Simulación básica

Técnica que consiste en recrear un proceso o sistema simple para analizar su comportamiento, sin incluir todos los detalles o variables complejas. Se utiliza para entender conceptos, probar ideas o validar modelos iniciales.

594. Simulación avanzada

Técnica que recrea un sistema o proceso complejo incluyendo múltiples variables, interacciones y condiciones realistas para obtener resultados precisos y detallados. Se utiliza para analizar comportamientos dinámicos, predecir escenarios y optimizar sistemas en entornos complejos.

595. Maqueta digital

Representación visual y funcional preliminar de una interfaz, sistema o producto digital creada mediante herramientas digitales. Sirve para mostrar la estructura, diseño y flujo antes del desarrollo final, facilitando la validación y retroalimentación.

596. Pixel

combinación de "picture" y "element". Elemento gráfico mínimo con el que se componen las imágenes en la pantalla de una computadora.

597. Diseño claro

se refiere a crear software con código fácil de leer, entender y mantener, utilizando buenas prácticas como la estructura organizada, nombres descriptivos, comentarios y un uso consistente de patrones de diseño, lo que resulta en aplicaciones más fiables y accesibles para otros programadores y para el propio mantenimiento futuro

598. Plantilla genérica

Documento o archivo base con una estructura estándar y flexible que puede adaptarse a diferentes proyectos, documentos o situaciones dentro del desarrollo de software. Está diseñada para ser reutilizable y modificable, facilitando la creación rápida y consistente de contenido sin estar limitada a un caso específico.

599. Plantilla personalizada

Documento o archivo creado o adaptado específicamente para un proyecto, equipo o necesidad particular dentro del desarrollo de software. Se diseña para ajustarse a requerimientos concretos, estilos, procesos o formatos únicos, facilitando un trabajo más alineado y eficiente.

600. Tema fijo

Configuración o diseño visual que mantiene un estilo, colores y apariencia constantes e invariables en una aplicación o sitio web. No permite cambios dinámicos por parte del usuario y garantiza una experiencia uniforme para todos los usuarios.

601. Tema dinámico

Configuración visual que permite cambiar o adaptar el estilo, colores y apariencia de una aplicación o sitio web en tiempo real, generalmente según las preferencias del usuario o condiciones del sistema. Ofrece una experiencia personalizada y flexible.

602. Lista corta

Es una selección reducida de elementos que han sido filtrados tras una evaluación inicial, y que cumplen con los criterios necesarios para una evaluación más profunda o toma de decisiones finales.

603. Lista extensa

Es una relación amplia de opciones, elementos o candidatos que se consideran en una etapa inicial de evaluación dentro de un proyecto de software esta lista contiene todas las alternativas posibles antes de aplicar filtros o criterios de selección.

604. Tabla reducida

es una versión simplificada o condensada de una tabla más grande y completa. Se utiliza para mostrar solo los datos más importantes, relevantes o necesarios para un propósito específico, ya sea durante la fase de análisis, diseño, pruebas o documentación de un sistema.

605. Tabla completa

es una representación estructurada de datos que contiene todos los campos, atributos y detalles necesarios de una entidad en un sistema es el modelo de datos definitivo y detallado que se utiliza para diseñar la base de datos en la etapa de desarrollo, basado en los requerimientos analizados previamente.

606. Vista previa

Es una función que permite ver cómo quedará un documento, diseño, página o aplicación antes de guardarlo o publicarlo de manera definitiva. En desarrollo de software se usa para comprobar el resultado de los cambios hechos (como diseño, texto o configuración) y asegurarse de que todo esté correcto.

607. Vista detallada

Es una representación dentro de un sistema o aplicación que muestra toda la información específica y completa de un elemento seleccionado. Se diferencia de una vista general o previa porque no solo enseña un resumen, sino que despliega atributos, características y datos más profundos del objeto, registro o componente.

608. Panel principal

Es la sección central de una aplicación o sistema, donde se muestran las funciones, opciones o contenidos más importantes. Generalmente, es la parte de la interfaz que concentra la mayor interacción del usuario, ya que desde allí se accede a las herramientas o módulos principales del software.

609. Beta tester

Un beta tester es un usuario de programas cuyos ejecutables están pendientes de terminar su fase de desarrollo, o alcanzar un alto nivel de funcionamiento, pero que aún no son completamente estables.

610. Ventana activa

la ventana activa se refiere a la interfaz o componente gráfico que actualmente tiene el control de la interacción del usuario dentro de un sistema. Es aquella que recibe directamente las entradas del teclado, los clics del ratón o los gestos táctiles, mientras que las demás ventanas permanecen abiertas, pero en un estado inactivo.

611. Ventana minimizada

es aquella que permanece en ejecución, pero deja de estar visible en la interfaz principal, quedando reducida a un ícono o a una barra de tareas. Aunque no esté en pantalla, la aplicación sigue activa en segundo plano y conservando su estado, lista para ser restaurada cuando el usuario lo necesite

612. Etiqueta visible

es un elemento de la interfaz gráfica de usuario (GUI) que muestra de manera clara un texto, símbolo o nombre con el fin de identificar, describir o guiar al usuario sobre el contenido o la función de un componente del sistema.

613. Etiqueta oculta

es un elemento de la interfaz gráfica o del código que no se muestra directamente al usuario en la pantalla, pero que cumple una función importante dentro del sistema

614. Campo obligatorio

es un elemento de entrada de datos que el usuario debe completar antes de poder avanzar o enviar un formulario. Se utiliza para garantizar que la información esencial del sistema se capture correctamente y evita errores o datos incompletos.

615. Campo opcional

Es un campo de entrada donde el usuario puede decidir si ingresa información o lo deja vacío. Su propósito es permitir flexibilidad, recolectando datos adicionales sin interrumpir el flujo principal de la aplicación.

616. Casilla marcada

Se refiere a un elemento de selección (checkbox) que ha sido activado por el usuario, indicando que una opción o condición se aplica. Es común en formularios y configuraciones donde se permiten múltiples selecciones.

617. Casilla vacía

Es la contraparte de la casilla marcada, indicando que la opción no ha sido seleccionada. Permite al usuario decidir qué elementos no se aplican o se omiten

618. Interruptor encendido

En software, un interruptor encendido (toggle switch activado) es un control de interfaz que indica que una función o configuración está activa. Se utiliza para alternar estados de manera clara e inmediata

619. Interruptor apagado

Es un interruptor que indica que la función o configuración está desactivada, permitiendo al usuario saber que la opción no está en uso en ese momento.

620. Deslizador corto

Es un control de interfaz que permite ajustar un valor en un rango limitado de manera rápida, ocupando poco espacio en la pantalla. Suelen utilizarse para configuraciones simples o ajustes rápidos.

621. Deslizador largo

Similar al deslizador corto, pero ocupa más espacio y generalmente permite ajustes más precisos o rangos más amplios. Facilita la selección detallada de valores dentro del sistema.

622. Opción única

Se refiere a un elemento de selección donde el usuario puede elegir solamente una opción de un conjunto, como en botones de radio (radio buttons). Garantiza que no haya selecciones conflictivas y es útil para decisiones exclusivas dentro del software.

623. Enlace interno

Es un vínculo dentro de un sistema, aplicación o sitio que dirige al usuario a otra sección o recurso dentro del mismo entorno. Facilita la navegación entre módulos o páginas relacionadas, mejorando la experiencia de uso y la organización de la información.

624. Enlace externo

Es un vínculo que lleva al usuario a un recurso o página fuera del sistema o aplicación, como un sitio web externo. Se usa para proporcionar información complementaria o referencial.

625. Hipervínculo válido

Un hipervínculo que funciona correctamente, llevando al usuario al destino esperado sin errores. Garantiza la confiabilidad y usabilidad de la navegación dentro de un sistema.

626. Hipervínculo roto

Es un enlace que no funciona y no lleva al destino esperado. Puede generar confusión, afectar la experiencia de uso y señalar problemas de mantenimiento o actualización dentro del sistema.

627. Archivo permanente

Es un archivo almacenado de manera definitiva dentro del sistema, diseñado para mantenerse intacto y contener información crítica o de referencia importante. Su integridad es clave para la seguridad y consistencia de los datos del software.

628. Copia restaurada

Es un archivo o conjunto de datos que ha sido recuperado a partir de un respaldo, devolviendo la información a un estado previo de manera segura. Permite recuperar datos perdidos o revertir cambios no deseados en el sistema.

629. Respaldo automático

Proceso mediante el cual el sistema guarda copias de seguridad de forma automática y periódica, protegiendo los datos ante fallos, pérdidas o cierres inesperados, sin necesidad de intervención del usuario

630. Respaldo manual

Proceso en el que el usuario decide cuándo y qué datos guardar, creando copias de seguridad bajo su control. Permite elegir de manera específica qué información proteger según las necesidades del sistema o proyecto.

631. Sincronización rápida

Es el proceso mediante el cual los datos entre sistemas o dispositivos se actualizan de manera casi inmediata, asegurando que la información se mantenga consistente en tiempo real. Este tipo de sincronización es útil cuando se requiere que los cambios estén disponibles de forma inmediata para todos los usuarios o módulos del sistema.

632. Sincronización lenta

Es la actualización de datos entre sistemas que toma más tiempo, generalmente debido al tamaño de los archivos o a limitaciones de conexiones. Aunque es más lenta, garantiza que todos los datos se alineen correctamente y se mantenga la integridad de la información

633. Migración parcial

Es el traslado de datos o componentes de un sistema a otro solo de manera selectiva, moviendo módulos o registros específicos sin afectar toda la estructura. Permite implementar cambios graduales y reducir riesgos en sistemas grandes o críticos.

634. Migración total

Es el traslado completo de todos los datos y componentes de un sistema a otro, asegurando que la información y la funcionalidad se mantengan íntegramente. Este tipo de migración suele requerir planificación para minimizar interrupciones en el uso del software.

635. Exportación de datos

636. Importación de datos

637. Carga inicial

638. Carga progresiva

Proceso de carga de datos que se realiza por etapas o bloques, optimizando los recursos del sistema y reduciendo el tiempo de espera del usuario. Facilita el acceso a la información mientras la carga completa se va completando en segundo plano.

639. Descarga rápida

640. Descarga lenta

Es la transferencia de datos que se realiza de forma más pausada, generalmente debido al tamaño de los archivos, limitaciones de la conexión o la capacidad del sistema. Aunque toma más tiempo, asegura que los datos se reciban de manera completa y correcta.

641. Guardado automático

Función del sistema que permite guardar información sin intervención del usuario, protegiendo los datos ante cierres inesperados, fallos del sistema o errores humanos. Esta función contribuye a la seguridad y continuidad de la operación dentro del software.

642. Guardado manual

643. Instalación básica

Proceso de instalación que incluye solo los componentes esenciales del software, suficientes para que la aplicación funcione correctamente. Permite un uso rápido y eficiente sin ocupar espacio o recursos adicionales innecesarios.

1282. Administración de memoria

Variedad de métodos utilizados para almacenar datos y programas en memoria, hacer un seguimiento de ellos y recuperar el espacio de memoria cuando ya no se necesitan. En los minicomputadores y mainframes tradicionales, la administración de memoria incluye la memoria virtual, la conmutación de bancos y las técnicas de protección de memoria.

1287. Controlador VXD

1289. X-Y-Z Matrix

Estructura tridimensional. Los ejes x e y representan las primeras 2 dimensiones, y el eje z, la tercera dimensión. En una imagen gráfica, la x y la y denotan el ancho y la altura, mientras que la z denota la profundidad.

1290. Full Duplex

Cualidad de los elementos que permiten la entrada y salida de datos de forma simultánea. El concepto está muy relacionado con el campo de las comunicaciones en vivo a través de la red, ya que indica que se puedo oír y hablar al mismo tiempo.

1291. Fuera de línea

1292. Editor
